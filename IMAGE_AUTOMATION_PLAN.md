# 이미지 자동화 업데이트 계획

## 목표
1. 기존에 생성된 Gemini 이미지(채팅 버블 내)에서 즉시 저장 기능 제공.
2. 새 프롬프트 전송 없이 현재 이미지 클릭 → 모달 열기 → "이미지 복사" 실행.
3. 저장 파일을 `generated_scripts/<storyId>/images/scene-XX_timestamp.png`에 기록.

## 진행 현황
- [x] `/api/image/ai-generate` 엔드포인트 추가 → Puppeteer와 프런트 간 통신 정비.
- [x] 멈춤(■) → 마이크 아이콘 감지 및 이미지 완성 시점 대기 로직 구현.
- [ ] 썸네일 클릭 후 모달에서 "이미지 복사" 버튼을 안정적으로 클릭하도록 DOM 셀렉터 재조정.
- [ ] 모달이 새 탭/레이어일 경우에 대비한 대체 셀렉터/탭 감지 추가.
- [ ] 복사 성공 시 정확한 경로에 저장하고, 실패 시 사용자에 대한 명확한 피드백 제공.

## 다음 단계
1. **모달 셀렉터 검증**: 실제 DOM 구조를 재확인하여 클릭 대상과 복사 버튼의 CSS 셀렉터를 업데이트합니다.
2. **클릭 안정화**: 클릭 전 조금 더 명확한 좌표 산정(예: bounding box 중심) 후 포커스 강제 부여.
3. **에러 로그 보강**: 실패 원인을 빠르게 파악할 수 있도록 세부 로그/토스트 메시지를 추가합니다.
4. **기능 분리**: "프롬프트 재전송형"과 "기존 이미지 저장형" 버튼을 UI에서 분리해 사용자 선택권 제공.
