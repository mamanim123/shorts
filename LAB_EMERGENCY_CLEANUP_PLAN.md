# 🚀 쇼츠랩 프롬프트 안정화 긴급 정리 계획 (v3.2)

## 1. 최종 목표
- 이미지 파트: `services/labpromptbuilder(백업).ts`의 고정 문구/체형/헤어 로직을 100% 복구
- 대본 파트: 말썽을 부리는 규칙을 핵심 규칙만 남겨 단순화
- 음성/립싱크: 필요한 구조만 남겨 오류/혼선 최소화

## 2. 작업 체크리스트

### [A] 이미지 로직 복구 (백업본 기반)
- [ ] `PROMPT_CONSTANTS`(START/END/NEGATIVE) 문구를 백업본으로 완전 원복
- [ ] `FEMALE_BODY`, `MALE_BODY` 체형 문구를 백업본으로 고정
- [ ] `buildLabImagePrompt`를 백업본 구조로 재정의

### [B] 대본 규칙 슬림화 (말썽 해결)
- [ ] POV 규칙 단순화: “화자는 자기 이름을 3인칭으로 부르지 말 것”만 유지
- [ ] 관계 명시 규칙 완화: “첫 등장 시 관계가 자연스럽게 드러나도록” 수준으로 조정
- [ ] 복선/반전 규칙 단순화: 장르 구조 안에 간단히 삽입해 인지 부하 축소

### [C] 음성/립싱크 구조 단순화
- [ ] `voiceType` 옵션 축소 검토 (예: `narration`, `dialog`)
- [ ] `lipSync` 추출 조건을 “작은따옴표 대사 있을 때만”으로 제한
- [ ] `narration.text` 쉼표 강제 규칙 제거

### [D] 프롬프트 모듈화 (가독성/유지보수)
- [ ] 거대한 프롬프트 문자열을 여러 블록으로 분리
- [ ] `BasePrompt`, `GenreGuide`, `CharacterRules` 등 조각 단위로 구성

## 3. 기대 효과
- 이미지 프롬프트 결과 품질 안정화
- 대본 생성 실패율 감소 (LLM 인지 부하 최소화)
- 기능 추가 시 유지보수 용이
