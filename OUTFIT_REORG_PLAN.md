# 의상 카테고리 정리/이동 + 장르별 선택 제어 플랜

## 목표
- 기본 의상(UNIFIED 리스트) 포함: 의상 **이동/삭제/추가** 관리 가능
- 장르 관리 모달 안에서 **장르별 허용 카테고리**를 선택
- 의상 랜덤 선택 시 **“선택한 카테고리만 사용”** 규칙 적용
- **불륜/외도(affair-suspicion)** 전용 규칙은 기존대로 유지

## 범위
- 쇼츠랩 입력탭 > 의상추출(의상 라이브러리)에서 이동/삭제 지원
- 장르 관리 모달에 장르별 의상 카테고리 선택 UI 추가
- 랜덤 의상 선택 로직에 장르별 카테고리 제한 적용

## 비범위
- 서버/DB 스키마 변경 없이 **프런트 로컬 저장** 기반으로 우선 구현
- 기존 LLM 선택 모드(의상 LLM 선택)의 출력 규칙 변경은 제외(필요 시 후속)

## 핵심 결정사항 (확정)
- 기본 의상(UNIFIED)도 **숨김/이동 가능**하도록 “오버라이드 레이어” 도입
  - 삭제 = 숨김(soft delete)
  - 숨김된 의상은 **랜덤 선택/목록 노출 모두 제외**
  - 이동 = 카테고리 오버라이드
- 장르별 카테고리 제한은 **허용 목록(allow list)** 형태로 저장
- affair-suspicion은 기존 SEXY 전용 규칙 유지 (카테고리 제한이 있어도 우선 규칙 적용)

## 데이터/저장 구조 (초안)
- 로컬 스토리지(appStorageService) 사용
  - `outfit-base-overrides`:
    - `{ hiddenBaseIds: string[], categoryOverrides: Record<baseId, categoryId> }`
  - `shorts-lab-genre-guidelines`에 `allowedOutfitCategories?: string[]` 확장

## UI 변경 (초안)
- 의상추출 탭
  - 기본 의상에도 “이동/숨김” 메뉴 제공
  - 사용자 의상은 기존 삭제 유지 + 카테고리 변경
- 장르 관리 모달
  - 장르 편집 영역에 “허용 카테고리” 멀티선택 추가
  - 선택 없을 때는 “전체 허용” 또는 “미설정” 표시

## 로직 변경 (확정)
- 의상 풀 구축 시:
  - 기본 의상에 오버라이드 적용 (숨김/카테고리 변경)
  - 사용자 의상과 병합 시 중복 처리 규칙 유지
- 랜덤 의상 선택(pickFemaleOutfit/pickMaleOutfit):
  - 장르별 허용 카테고리 존재 시 그 카테고리만 필터
  - affair-suspicion은 SEXY 우선 규칙 유지

## 마이그레이션/호환
- 기존 저장 데이터가 없어도 안전한 기본값 제공
- 오버라이드/장르 제한이 없으면 기존 동작 유지

## 테스트 체크리스트
- 기본 의상 숨김 시 라이브러리/랜덤 선택에서 제외되는지
- 기본 의상 카테고리 이동 시 해당 카테고리로 노출되는지
- 사용자 의상 이동/삭제 정상 동작
- 장르별 카테고리 제한 적용 여부
- affair-suspicion 장르에서 SEXY 규칙 유지 확인
