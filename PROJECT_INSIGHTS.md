# 💡 프로젝트 인사이트 (Project Insights)

## 📌 기술적 결정 및 발견 (Technical Insights)

### 1. 폴더 구조 및 정렬 (2026-01-18)
- **결정**: 폴더명을 `YYMMDD_HHMM_Title`로 변경.
- **이유**: 단순 시퀀셜 번호(`-2`, `-3`)는 대량 생성 시 혼란을 야기함. 타임스탬프는 고유성을 보장하며 생성 순서대로 자동 정렬되는 이점이 있음.
- **발견**: 서버 측에서 `mtimeMs`로 정렬하여 불러오는 로직을 병행해야 사용자가 가장 최근 작업을 즉시 확인할 수 있음.

### 2. AI 응답 파싱 복구력 (2026-01-18)
- **결정**: 3단계 파싱 전략 (Brace matching -> Code block -> Whole text repair) 및 정규식 Fallback 도입.
- **이유**: LLM이 JSON 뒤에 불필요한 설명이나 `#` 등의 문자를 붙이는 경우가 잦음. 표준 파싱만으로는 폴더 생성 실패율이 높음.
- **발견**: JSON이 완전히 깨졌더라도 `title` 필드만 정규식으로 뽑아낼 수 있다면, 최소한 유효한 폴더명을 유지하며 저장이 가능함.

### 3. 프롬프트 엔진 파편화 해결 (2026-01-18)
- **결정**: 모든 프롬프트 핵심 지침을 `.md` 파일(`script_engine.md`, `visual_engine.md`)로 분리 및 `promptBuilder.ts` 통합.
- **이유**: 소스 코드(TSX, TS) 내에 섞인 수천 줄의 프롬프트는 유지보수가 불가능하며 AI에게 일관된 지시를 내리기 어려움.
- **발견**: 엔진 지침을 최상위에 배치하고 하드코딩된 규칙과 충돌을 제거함으로써, LLM의 지시 준수율(Instruction Following)이 현저히 향상됨.

## 📈 품질 향상 전략 (Quality Insights)

### 1. 캐릭터 네이밍 (Character Naming)
- '김여사', '김프로'와 같은 정형화된 호칭은 시청자의 몰입을 방해함.
- 한국에서 흔히 쓰이는 구체적인 이름(지영, 미숙, 준호)을 사용하고 상황에 맞는 호칭(언니, 씨, 프로님)을 조합하는 것이 훨씬 전문적인 스크립트로 느껴짐.

### 2. 반전(Twist)의 미학
- 쇼츠의 끝에서 상황을 설명하지 말고 '보여줘야' 함. 
- "알고 보니 A는 B였다"는 이미 실패한 스크립트. A의 짧은 행동 하나로 시청자가 스스로 "아!" 하고 깨닫게 만드는 것이 바이럴 포인트.
