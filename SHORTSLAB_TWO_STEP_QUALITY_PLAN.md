# ShortsLab 2단계 생성 품질 향상 계획

## 목표
- 2단계 생성에서 **AI대본생성보다 더 높은 품질**의 대본/이미지프롬프트를 만든다.
- 대본 생성과 이미지프롬프트 생성을 분리하여 각 단계에 **집중 최적화**를 적용한다.
- 기존 AI대본생성의 강제 규칙(고정 문구, 체형 문구, 슬롯 일관성)을 **2단계에도 100% 강제**한다.

## 문제 정의 (현재)
- 2단계는 별도 로직으로 인해 품질이 AI대본생성보다 낮음.
- 캐릭터 추출/샷 분배/체형 문구 삽입이 불안정.
- LLM이 씬 분해 JSON을 잘못 반환할 때 대응이 약함.

## 개선 방향
1) **대본 생성 품질 강화**
   - 훅/배경/전개/반전 순서 규칙 강제
   - 캐릭터가 드러나는 문장 구조 유도

2) **이미지프롬프트 품질 강화**
   - AI대본생성의 프롬프트 규칙/고정 문구 100% 재사용
   - 모든 여성에 체형 고정 문구 의무 삽입
   - 원샷에서도 identity/hair/body/outfit 강제 삽입

3) **캐릭터/샷 안정화**
   - 대본 기반 캐릭터 추출 결과를 우선 적용
   - characterIds 누락 시 재시도/실패 처리
   - 투샷/쓰리샷 최소 비율 룰 적용

## 체크리스트

- [x] 이미지프롬프트 생성에 AI대본생성 고정 규칙 재사용
- [x] 모든 여성 체형 문구 강제 삽입(원샷 포함)
- [x] 캐릭터 슬롯 매핑 안정화(WomanA/B/C/D 포함)
- [x] 샷 분배 최소 비율(원샷/투샷/쓰리샷) 적용
- [ ] LLM 씬 분해 JSON 실패 시 재시도/중단 처리
