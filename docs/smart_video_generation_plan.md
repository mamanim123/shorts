# 🎬 스마트 씨네보드 동영상 생성 구현 계획서 (v1.0)

## 1. 개요
현재 씨네보드의 동영상 생성 로직은 단순히 이미지용 프롬프트만을 사용하여 정적인 영상을 생성함. 이를 개선하여 **대본의 맥락(감정, 대사, 행동)**이 반영된 "스마트"한 영상을 생성하고, **Google Veo 3.1**을 인증 기반으로 연동하여 고품질의 결과물을 얻는 것을 목표로 함.

## 2. 현재 문제점 (As-Is)
- **서비스 불일치**: GenSpark(텍스트 AI)을 비디오 생성기로 오용 중.
- **맥락 결여**: 씬별 대본(Script) 내용을 무시하고 이미지 프롬프트만 사용.
- **인증 부재**: 고품질 영상 생성을 위한 Google 계정 인증 연동 미비.
- **일관성 부족**: 이전 씬 이미지와의 연결성이 고려되지 않음.

## 3. 개선 방향 (To-Be)

### 3.1 스마트 프롬프트 엔진 (Context-Aware)
단순 프롬프트 전달이 아닌, LLM을 거쳐 **"비디오 전용 동적 프롬프트"**를 생성.
- **입력**: 이미지 프롬프트 + 씬 대본 + 인물 감정 + 전후 맥락
- **추출 항목**: 
  - 핵심 동작 (Action): 예) "커피잔을 손으로 감싸며 부들부들 떪"
  - 표정 변화 (Facial Expression): 예) "처음엔 무표정하다가 서서히 눈물을 흘림"
  - 카메라 워킹 (Dynamic Camera): 예) "인물의 눈으로 천천히 줌인"

### 3.2 Google Veo 3.1 인증 기반 연동
- **방식**: Puppeteer를 통한 구글 로그인 세션 유지 (VideoFX 또는 AI Studio 활용).
- **장점**: API Key 제한 우회, 고해상도 생성, 안전 필터링 유연 대응.
- **구현**: `puppeteerHandler.js`에 `VIDEOFX` 서비스 프로필 추가.

### 3.3 Image-to-Video (I2V) 워크플로우
- 이미 생성된 **인물/장면 이미지**를 영상 생성의 '시작 프레임'으로 참조.
- **일관성**: 씨네보드에서 생성한 인물의 얼굴과 복장이 영상에서도 그대로 유지됨.

## 5. 기대 효과
- **몰입감 증대**: 대본의 대사와 인물의 움직임이 일치하여 쇼츠 제작 효율 극대화.
- **고품질 유지**: 최신 Veo 3.1 모델 사용으로 상업적 수준의 영상미 확보.
- **완전 자동화**: 클릭 한 번으로 "대본 해석 -> 이미지 생성 -> 맥락 반영 영상" 완성.

## 6. 스마트 비디오 프롬프트 편집 UI/UX 상세 설계

### 6.1 프롬프트 관리 프로세스 (Step-by-Step)
1. **분석 및 제안**: 사용자가 장면의 [영상] 탭을 클릭하거나 [프롬프트 생성] 버튼을 누르면, 서버의 LLM(Gemini 2.0 Flash)이 해당 장면의 대본과 이미지 프롬프트를 분석하여 **비디오 전용 동적 지시어**를 생성함.
2. **UI 표시**: 생성된 비디오 프롬프트는 각 장면 카드 내 [영상] 탭의 편집창(Textarea)에 즉시 표시됨.
3. **사용자 편집**: 사용자는 화면에 표시된 비디오 프롬프트를 직접 검토하고, 필요에 따라 동작, 카메라 워킹, 특수 효과 등을 자유롭게 수정할 수 있음.
4. **최종 생성**: 사용자가 수정된 프롬프트를 확정하고 [비디오 생성] 버튼을 클릭하면, **수정된 최종 텍스트**가 Puppeteer를 통해 Google VideoFX(Veo)에 전달되어 영상이 제작됨.

### 6.2 UI 구성 요소
- **Prompt Editor**: 실시간 편집이 가능한 다중 라인 입력창 (Textarea).
- **Generation Control**: 프롬프트 분석 버튼 및 비디오 실제 생성 버튼 분리.
- **Status Indicator**: 프롬프트 생성 중, 비디오 렌더링 중 등 단계별 상태 표시.

### 6.3 기술적 보완 사항
- **API 분리**: `/api/video/refine-prompt` (프롬프트 생성 전용) 및 `/api/video/generate-smart` (확정 프롬프트 기반 생성)로 기능을 나눔.
- **안정성**: Puppeteer의 요소 대기(Wait for selector) 타임아웃 및 예외 처리를 강화하여 500 에러 방지.
