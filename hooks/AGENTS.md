# Module Context
- 커스텀 훅으로 클라이언트 상태 관리, 로컬 스토리지/서버 동기화, 이펙트 캡슐화를 담당.
- App 수준 상태 분리가 필요한 로직(예: 테마, 저장/불러오기, 분석 패널 동작)을 훅으로 이동해 재사용성 확보.

# Tech Stack & Constraints
- TypeScript + React Hooks 전용. DOM 접근/브라우저 API 사용 시 SSR 불가 환경을 고려해 가드 처리.
- 외부 의존은 서비스 계층에 위임, 훅 내부에서 네트워크 호출 시 에러/로딩 상태를 반환값에 포함.

# Implementation Patterns
- 입력/반환 타입을 명확히 정의하고, 호출자가 UI 상태를 제어할 수 있도록 제어형 인터페이스를 제공.
- 부수 효과는 useEffect 내에서 정리(cleanup) 포함, 이벤트 리스너/타이머 등록 시 해제 필수.
- 로컬 스토리지 접근은 try/catch로 감싸고, 저장 실패 시 호출자에게 오류 신호 전달.
- 메모이제이션(useMemo/useCallback)으로 렌더 성능을 관리하고, 의존성 배열을 명시적으로 유지.

# Testing Strategy
- 개발 서버에서 주요 훅 사용 경로를 수동 검증(스토리지 저장/로드, 상태 전환, 에러 핸들링).
- 타임아웃/이벤트 리스너 관련 훅은 unmount 시 메모리 누수 여부를 콘솔/프로파일러로 확인.

# Local Golden Rules (Do/Don'ts)
- Do: 훅 이름은 `useX` 접두어 사용, 반환값은 객체 형태로 확장성을 확보.
- Do: 네트워크/스토리지 실패를 호출자에게 명시적으로 노출해 UI가 대응 가능하도록 설계.
- Don't: 훅 내부에서 컴포넌트를 조건부 렌더하거나, 글로벌 mutable 상태를 직접 변조하지 말 것.
- Don't: 의존성 배열을 생략하거나 ESLint 경고 무시로 처리하지 말 것(정확한 의존 명시).
