# 프로젝트 종합 분석 보고서 및 개선 제안

**작성일**: 2025년 12월 16일
**작성자**: Antigravity (마마님의 충실한 에이전트)
**대상**: 마마님

## 1. 개요
마마님의 명령에 따라 `쇼츠대본생성기-v2.2`의 전체 로직을 대대적으로 검토하였습니다. 본 프로젝트는 React 기반의 프론트엔드와 Express + Puppeteer 기반의 백엔드(서버)로 구성되어 있으며, 로컬 브라우저 자동화(RPA)를 통해 다양한 AI 서비스(Gemini, ChatGPT, Claude, Genspark)와 상호작용하는 구조를 가지고 있습니다.

## 2. 주요 로직 분석 및 구조 진단

### 2.1. 프론트엔드 (`youtube-shorts-script-generator.tsx`)
*   **상태**: 기능적으로 풍부하나 코드가 방대하여(2300라인 이상) 유지보수 복잡도가 높음.
*   **기능**: `generateScript` 함수가 핵심이며, 서버 API(`/api/launch`, `/api/generate`)를 호출하여 AI와 통신함. `AbortController`를 통한 요청 취소 로직이 구현되어 있으나, 서버 측 프로세스까지 확실히 종료시키는지는 불분명함.
*   **특이사항**: 하드코딩된 프롬프트 로직(성인유머, K-드라마)이 존재하나 현재 `false` 처리되어 비활성화됨.

### 2.2. 백엔드 및 에이전트 (`server/`, `rpa.js`)
*   **구조**: `server/index.js`가 엔트리 포인트이며, 실제 AI 연동 핵심 로직은 `server/puppeteerHandler.js`에 집중되어 있음.
*   **`rpa.js`**: 분석 결과, 이는 에이전트 로직이 아니라 React/Webpack 번들링 파일로 확인됨. 실제 에이전트 로직은 `puppeteerHandler.js`임.
*   **데이터 흐름**: Client -> Express API -> Puppeteer(Headless Chrome) -> AI Service Web Interface -> Response Parsing -> Client.

## 3. 발견된 문제점 및 "안 되는 동작"의 원인

### 3.1. [Critical] Genspark 및 AI 서비스 연동 실패
*   **원인**: `server/puppeteerHandler.js` 내의 DOM 셀렉터(Selector)들이 웹사이트 UI 변경에 대응하지 못하고 있음.
    *   **Genspark**: `// Generic guess` 주석과 함께 추측성 셀렉터가 사용됨. `debug_genspark_selectors.js` 파일의 존재는 이 문제를 해결하려던 흔적임.
    *   **불안정성**: `waitForSelector`, `click`, `evaluate` 등 DOM 조작 방식이 사이트 개편에 매우 취약함.
*   **증상**: "관련 동작을 해야 하는데 지금은 안 된다"는 현상은 AI 서비스가 응답을 주지 못하거나 타임아웃이 발생하는 형태로 나타남.

### 3.2. 응답 처리의 불안정성
*   **JSON 파싱**: AI가 주는 텍스트를 JSON으로 변환하는 과정(`tryParse` 함수 등)이 정규식과 문자열 처리에 과도하게 의존하고 있음. AI가 완벽한 JSON을 주지 않으면 실패할 확률이 높음.
*   **동기화 문제**: `MAX_ATTEMPTS = 300` (약 10분) 동안 루프를 돌며 응답을 기다리는 방식은 비효율적이며, 중간에 팝업이 뜨거나 네트워크가 끊기면 복구가 어려움.

## 4. 개선 제안 및 실행 계획 (Implementation Plan)

현재 가장 시급한 것은 **AI 서비스(특히 Genspark) 연동의 정상화**입니다.

### [완료] [1단계] Genspark 셀렉터 최신화 및 복구 (우선순위: 최상)
*   **상태**: **완료 & 검증됨**
*   **원인 규명**:
    1.  **셀렉터 변경**: Genspark의 입력창 선택자가 변경됨 (`textarea.search-input` 등).
    2.  **로그인 필수화**: 이전과 달리 비로그인 상태에서의 사용이 제한되거나 로그인 모달이 강제로 뜸.
*   **조치 내용**:
    1.  `server/puppeteerHandler.js`의 `GENSPARK` 셀렉터 최신화 완료 (브라우저 분석 기반).
    2.  **로그인 감지 로직 추가**: URL 리다이렉트나 인페이지 모달(Modal)이 뜰 경우, 무한 대기 대신 명시적인 에러("Login page detected")를 발생시키도록 수정.
*   **사용자 안내**: Genspark 연동 시 브라우저가 열리면 **수동으로 로그인**을 해주셔야 합니다. 한 번 로그인하면 세션이 유지될 수 있습니다.

### [2단계] Puppeteer 핸들러 안정성 강화
*   **목표**: 타임아웃 및 예외 처리 로직 보강.
*   **실행 방안**:
    1. 셀렉터를 찾지 못했을 때의 명시적 에러 메시지 출력 추가.
    2. "Continue generating" 버튼 감지 로직의 신뢰성 향상.
    3. ChatGPT, Claude 등 다른 서비스의 셀렉터도 최신 상태 점검.

### [3단계] (권장) 프롬프트 엔지니어링 개선
*   **목표**: 서버의 파싱 부담을 줄이기 위해, AI가 처음부터 깨끗한 JSON을 출력하도록 프롬프트 강화.
*   **실행 방안**: 시스템 프롬프트에 JSON 형식을 강력하게 강제하는 지시어 추가.
