# 20251222_샤베트프롬프트분석_plan

## 목표
마마님께서 "샤베트 의상/헤어" 엔진 및 "연령" 선택 후 대본 생성 시, 의도치 않은 프롬프트가 개입되는 현상을 분석하고 그 원인과 흐름을 보고한다.

## 작업 순서

1.  **프론트엔드 분석 (`youtube-shorts-script-generator.tsx`)**
    *   `Generate` 버튼 클릭 시 호출되는 함수 식별.
    *   선택된 `engine` (샤베트 등)과 `age` (연령) 값이 API 페이로드에 어떻게 담기는지 확인.
    *   하드코딩된 프롬프트 전달 여부 확인.

2.  **백엔드 분석 (`server` 및 `services`)**
    *   `/api/generate` (또는 관련) 엔드포인트 수신부 확인.
    *   받은 파라미터를 사용해 LLM(OpenAI/Gemini/Claude) 서비스 호출 시 프롬프트를 구성하는 로직 추적.
    *   `system` 프롬프트와 `user` 프롬프트가 합쳐지는 과정에서 별도의 'Injection'이 있는지 확인.

3.  **데이터 파일 확인**
    *   `샤베트`라는 용어가 정의된 상수 파일이나 설정 파일 탐색 (`ImageStudioData.ts`, `constants.ts` 등).
    *   해당 정의에 숨겨진 추가 프롬프트 텍스트가 있는지 확인.

4.  **결과 보고**
    *   프롬프트가 최종적으로 어떻게 완성되어 LLM에 전달되는지 단계별 설명.
    *   문제의 "끼어드는 프롬프트"가 어디서 유래했는지 특정.
