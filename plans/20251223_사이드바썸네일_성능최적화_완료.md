**작성일**: 2025-12-23

## 🔴 발견된 근본 문제

### **문제 1: 무한 이미지 리로딩**
```typescript
// BEFORE - 문제 코드
useEffect(() => {
  loadUrls();  // imageHistory 변경될 때마다 모든 이미지 재로드!
}, [imageHistory, historyUrls]);  // historyUrls도 의존성이라 무한 루프
```

**증상**:
- 페이지 이동할 때마다 모든 썸네일이 계속 로딩 중
- 이미 로드된 이미지도 반복해서 재로드
- 브라우저 리소스 미친듯이 소모
- 결국 브라우저 크래시

### **문제 2: 디바운스 없음**
- `imageHistory`가 조금만 변경되어도 즉시 전체 로딩 시작
- 연속적인 상태 변경 시 불필요한 중복 로딩

### **해결책 2: 디바운스 적용 (300ms)**

```typescript
// AFTER - 디바운스 추가
const loadUrlsTimeoutRef = useRef<NodeJS.Timeout | null>(null);

useEffect(() => {
  if (loadUrlsTimeoutRef.current) {
    clearTimeout(loadUrlsTimeoutRef.current);
  }

  loadUrlsTimeoutRef.current = setTimeout(() => {
    loadUrls(); // 300ms 후 실행
  }, 300);

  return () => {
    if (loadUrlsTimeoutRef.current) {
      clearTimeout(loadUrlsTimeoutRef.current);
    }
  };
}, [imageHistory]); // historyUrls 의존성 제거!
```

## 🧪 테스트 방법

### **콘솔에서 확인할 로그**

1. 페이지 첫 로딩:
   ```
   [Performance] Loading 6 new thumbnails (0 already loaded)
   [Performance] Loaded 6 thumbnails successfully
   ```

2. 다른 탭으로 이동 후 돌아오기:
   ```
   (로그 없음 - 이미 로드됨!)
   ```

3. 새 이미지 생성 시에만:
   ```
   [Performance] Loading 1 new thumbnails (6 already loaded)
   [Performance] Loaded 1 thumbnails successfully
   ```

### **테스트 시나리오**

#### **시나리오 1: 쇼츠 생성기 탭**
1. "쇼츠 생성기" 탭 클릭
2. 콘솔 확인: 처음 한 번만 로딩 로그 출력
3. 다른 탭 갔다가 돌아오기
4. 콘솔 확인: **로딩 로그 없어야 함** ✅

#### **시나리오 2: 보관함 - 생성대본**
1. "보관함" → "생성대본" 클릭
2. 대본 하나 클릭
3. 콘솔 확인: **로딩 로그 없어야 함** ✅

#### **시나리오 3: 보관함 - 영상대본**
1. "영상대본" 탭 클릭
2. 대본 클릭
3. 콘솔 확인: **로딩 로그 없어야 함** ✅

#### **시나리오 4: 보관함 - 테스트대본**
1. "테스트대본" 탭 클릭
2. 테스트 대본 클릭
3. 콘솔 확인: **로딩 로그 없어야 함** ✅

#### **시나리오 5: 빠른 탭 전환 (디바운스 테스트)**
1. 쇼츠 생성기 → 보관함 → 쇼츠 생성기를 빠르게 반복 (1초 안에)
2. 콘솔 확인: 로딩이 한 번만 실행되어야 함 (300ms 디바운스) ✅
