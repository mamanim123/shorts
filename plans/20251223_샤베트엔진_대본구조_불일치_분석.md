# 샤베트 엔진 대본 구조 불일치 문제 분석

**문제**: 샤베트 엔진이 역할 섹션에 명시된 대본 구조를 따르지 않음

---

## 🔍 발견된 문제

### 불일치하는 두 가지 구조 정의

**1. ## 1. 역할 섹션 (Mama-nim이 원하는 구조)**:
```
목표는 "강력한 훅 -> 배경및 상황 설명 -> 달달한 설렘 → 엉뚱한 오해 → 극적인 상황 -> 귀여운 반전"
```

**2. ## 3. 이야기 구조 섹션 (AI가 실제로 따르는 구조)**:
```
1. Hook: 엉뚱 선언/속닥속닥 비밀/직접 질문으로 바로 갈등을 던진다.
2. Spark: 상황 설명은 한 줄, 곧바로 엉뚱한 행동과 의성어·의태어를 섞어 tension을 올린다.
3. Melt: 대사와 나레이션을 번갈아 배치(대사 ≥ 10%). 두근거림, 허당 포인트, 웃긴 TMI를 교차시킨다.
4. Twist: 1문장으로 귀여운 진실을 공개하고, 즉시 리액션 1문장으로 마무리한다.
```

### 왜 이런 일이 발생하는가?

AI 모델은 **더 구체적이고 상세한 지시**를 우선적으로 따릅니다:
- "## 1. 역할" 섹션: 추상적인 목표 설명
- "## 3. 이야기 구조" 섹션: 구체적인 단계별 지시

→ AI는 "## 3. 이야기 구조"의 구체적인 지시를 따름

---

## 💡 해결 방안

### Option A: "## 3. 이야기 구조" 섹션을 Mama-nim의 구조로 교체 (권장)

**변경 전**:
```
1. Hook
2. Spark
3. Melt
4. Twist
```

**변경 후**:
```
1. **강력한 훅**: 시청자를 멈추게 만드는 강력한 도발/질문으로 시작
2. **배경 및 상황 설명**: 시간·장소·상황을 한 문장으로 빠르게 고정
3. **달달한 설렘**: 두근거림, 기대감, 미묘한 감정을 표현
4. **엉뚱한 오해**: 상황을 오해하게 만드는 행동이나 대사
5. **극적인 상황**: 긴장감을 최고조로 끌어올림
6. **귀여운 반전**: 1문장으로 진실을 공개하고 즉시 리액션으로 마무리
```

### Option B: "## 1. 역할" 섹션 삭제 (비권장)

역할 섹션의 목표 설명을 삭제하고 "## 3. 이야기 구조"만 유지
