## 목표
- 샤베트 코미디 v1.3 대본 생성 시 주인공 의상 1개만 랜덤 확정해 LLM 프롬프트에 전달하고, 전체 의상 리스트 전송으로 인한 토큰 낭비를 제거한다.

## 작업 범위
- 프런트 서비스 계층(`services/geminiService.ts` 등)에서 프롬프트 조립 로직을 수정해 의상 전달을 슬림화한다.
- 엔진 프롬프트 로딩/개인화 과정에서 불필요한 의상 리스트 삽입을 차단한다.

## 구현 단계
1) 현행 흐름 파악: 사용자 입력 → 엔진 프롬프트 로딩 → 최종 프롬프트 조합 지점에서 의상 리스트가 포함되는 경로를 확인한다.
2) 주인공 의상 결정: 주인공용 의상을 1개 랜덤 선택(또는 사용자 지정값 우선) 후 그 값만 프롬프트에 삽입하도록 로직을 수정한다.
3) 서브/남성 의상 규칙: 서브/남성 캐릭터는 “주인공과 겹치지 않는 유사 스타일 자동 선택”이라는 간단 지시문만 남기고 리스트 본문 전송을 제거한다.
4) 프롬프트 슬림화: 샤베트 엔진 시스템 프롬프트에 포함된 의상 리스트 블록이 최종 요청에 전달되지 않도록 필터링하거나 축약한다.
5) 검증: 샘플 입력으로 최종 전송 프롬프트를 확인해 주인공 의상 1개만 노출되는지, 의상 리스트 전체가 제거됐는지 점검한다.

## 주의사항
- 프롬프트 구조/필수 규칙은 유지하되, 의상 리스트 본문만 제거/축약한다.
- 타입 안전성 유지(TypeScript), 기존 API 계약(문자열 스키마) 변경 금지.
- 토큰 절감 외 로직 변경은 최소화한다.
