# 이미지 히스토리 사이드바 중복 키 및 썸네일 미표시 수정 계획

## 1. 문제 분석
- **중복 키 오류**: `bootstrapFromDisk` 함수가 `Date.now()`를 기반으로 `id`를 생성하는데, React Strict Mode나 빠른 연 호출로 인해 동일한 밀리초에 여러 번 호출되면서 중복된 `id`가 생성됨.
- **썸네일 미표시**: 
    - 중복 키로 인해 React 렌더링이 깨짐.
    - `loadUrls` 함수에서 `historyUrls` 업데이트 시 레이스 컨디션 발생 가능성.
    - 새로 생성된 이미지의 경우 `setImageHistory`가 함수형 업데이트를 사용하지 않아 상태가 덮어씌워질 수 있음.

## 2. 수정 사항

### 2.1 `youtube-shorts-script-generator.tsx` 수정
1.  **`bootstrapFromDisk` 개선**:
    - `id` 생성 방식을 `disk-${filename}`으로 변경하여 파일명 기반의 고유하고 안정적인 ID 보장.
    - `setImageHistory` 호출 시 기존 `id`와 중복되지 않도록 필터링 로직 추가.
2.  **`handleGenerateImage` 개선**:
    - `setImageHistory`를 함수형 업데이트(`prev => ...`)로 변경하여 상태 일관성 유지.
3.  **데이터 로드 시 중복 제거**:
    - `useEffect`에서 `localStorage` 데이터를 불러올 때 `id` 기준 중복 제거 로직 추가.
4.  **`loadUrls` 개선**:
    - 이미 로딩 중이거나 로딩된 `id`는 건너뛰도록 최적화.

### 2.2 `ShortsImageHistorySidebar.tsx` 확인 (필요 시)
- 현재 `item.id`를 키로 사용하고 있으므로, 데이터 쪽에서 `id` 중복만 해결하면 됨.

## 3. 기대 효과
- 콘솔의 `Encountered two children with the same key` 오류 해결.
- 이미지 생성 후 사이드바에 즉시 썸네일이 표시됨.
- 디스크 이미지 새로고침 시 중복 항목이 생기지 않음.
