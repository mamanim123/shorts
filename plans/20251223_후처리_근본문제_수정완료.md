## 💥 문제 분석

### 기존 로직 (Line 1240-1246)
```typescript
const processedScripts = postProcessScripts(result.scripts, {
  femaleOutfit: useDynamicOutfit ? undefined : lockedFemaleOutfit,
  femaleOutfit2: useDynamicOutfit ? undefined : lockedFemaleOutfit2,
  maleOutfit: lockedMaleOutfit,
  autoEnhance: activeEnhancementSettings.autoEnhanceOnGeneration ?? true, // ← 문제!
  enhancementSettings: activeEnhancementSettings
});
```

**문제점**:
1. ❌ `postProcessScripts`는 **항상 호출됨**
2. ❌ `autoEnhance` 값만 전달할 뿐, 호출 여부를 결정하지 않음
3. ❌ 기본값이 `true` → 설정 로드 실패 시 자동으로 후처리 실행
4. ❌ 후처리 OFF여도 `postProcessScripts` 함수는 실행되고, 내부에서 `enhanceScenePrompt` 호출

### 결과
- 사용자가 후처리 OFF 설정
- 하지만 `postProcessScripts`는 계속 실행
- `enhanceScenePrompt`도 계속 호출
- 의상 추가 로직 실행 → 중복 발생

## ✅ 해결 방법 (Option 1)

### 수정 1: 대본 생성 시 (Line 1240-1252)
```typescript
// [FIX] Only call postProcessScripts if autoEnhance is enabled
// If post-processing is OFF, use LLM output directly without modification
const shouldPostProcess = activeEnhancementSettings.autoEnhanceOnGeneration ?? false; // Default to false for safety
const processedScripts = shouldPostProcess 
  ? postProcessScripts(result.scripts, {
      femaleOutfit: useDynamicOutfit ? undefined : lockedFemaleOutfit,
      femaleOutfit2: useDynamicOutfit ? undefined : lockedFemaleOutfit2,
      maleOutfit: lockedMaleOutfit,
      autoEnhance: true, // Already checked above
      enhancementSettings: activeEnhancementSettings
    })
  : result.scripts; // Use LLM output as-is
```
