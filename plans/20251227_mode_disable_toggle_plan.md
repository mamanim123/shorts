# 생성 모드 비활성화 토글 추가 계획

1. **UI 버튼 설계**  
   - "생성 모드 선택" 섹션에 활성/비활성 상태를 표시하는 토글 버튼(예: "모드 비활성화")을 추가한다.  
   - 토글 켜짐 시 모드 카드 버튼을 비활성화하고, 사용자에게 "엔진 프롬프트만 사용" 안내 배지를 보여준다.

2. **상태·스토리지 연동**  
   - `useState`로 `isModeDisabled` 상태를 추가하고 localStorage(`shorts-generator-mode-disabled` 등)에 저장/복원한다.  
   - 활성화 시 기존 `generationMode` 선택은 유지하되, 실제 프롬프트 구성 단계에서 토글 상태를 체크하도록 한다.

3. **프롬프트 생성 로직 변경**  
   - `generateScript`에서 토글이 켜져 있을 경우, `promptBody`를 엔진 프롬프트만 사용하도록 변경한다. (즉, modeTemplates 사용 X)  
   - 프리셋 사용 시 기존 동작 유지, 토글이 켜져도 프리셋이 최우선이 되도록 명시한다.

4. **UX/메시지 보강 및 테스트**  
   - 모드 비활성화 시 토스트 또는 텍스트로 "엔진 지침만 사용" 상태를 알려주고, 재활성화 UX 마련.  
   - 수동 테스트: 토글 온/오프 후 대본 생성 → 프롬프트 로그 확인하여 mode template가 제거되는지 검증한다.
