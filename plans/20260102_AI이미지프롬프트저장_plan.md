# Implementation Plan - AI 생성 이미지 프롬프트 내장 저장 및 복구

## 1. 개요
- **작업 목표**: AI로 생성된 이미지 파일(PNG) 내부에 원본 프롬프트를 메타데이터로 내장하고, AI Studio에서 이를 복원하여 '프롬프트 복사' 기능이 정상 동작하도록 한다.
- **요청자**: 마마님
- **작성일**: 2026-01-02

## 2. 현상 분석
- 현재 이미지 저장 시 원본 프롬프트 정보가 유실되어, 파일명에 의존하고 있다.
- 별도의 텍스트 파일 생성 없이 이미지 파일 자체에 정보를 담는 '스마트한' 방식이 필요하다.

## 3. 수정 계획

### 3.1. 서버 수정 (`server/index.js`)
1.  **PNG 메타데이터 유틸리티 함수 구현**
    - `addTextChunk(buffer, key, text)`: PNG 버퍼에 `tEXt` 청크를 추가하는 함수.
    - `extractTextChunk(buffer, key)`: PNG 버퍼에서 특정 키의 `tEXt` 청크 값을 읽어오는 함수.
    - 외부 라이브러리 없이 순수 Node.js Buffer 조작으로 구현.

2.  **이미지 저장 로직 수정**
    - `/api/image/ai-generate` 및 `/api/save-image` 엔드포인트.
    - 이미지 데이터를 파일로 쓰기 전에 `addTextChunk`를 호출하여 `Prompt` 키로 원본 프롬프트를 삽입한다.

3.  **이미지 목록 조회 API 수정**
    - `/api/images/by-story/:storyId` 엔드포인트.
    - 이미지 파일을 읽을 때 `extractTextChunk`를 호출하여 `Prompt` 값을 읽어온다.
    - 읽어온 프롬프트를 반환 객체의 `prompt` 필드에 담는다.

### 3.2. 클라이언트 수정 (`components/OutputDisplay.tsx`)
1.  **원격 이미지 목록 처리 로직 수정**
    - `fetchRemoteImagesForStory` 함수 수정.
    - 서버에서 받은 `prompt` 값을 사용하여 `ImageHistoryItem`을 생성한다.

// PNG Signature: 89 50 4E 47 0D 0A 1A 0A
function addTextChunk(buffer, key, text) {
    if (!text) return buffer;
    // tEXt chunk format: Length (4 bytes) + Type (4 bytes, "tEXt") + Data (Key + \0 + Text) + CRC (4 bytes)
    const keyBuffer = Buffer.from(key, 'latin1');
    const textBuffer = Buffer.from(text, 'latin1'); // PNG tEXt supports Latin-1
    const dataLen = keyBuffer.length + 1 + textBuffer.length;
    const chunkLen = 4 + 4 + dataLen + 4;
    
    const chunk = Buffer.alloc(chunkLen);
    chunk.writeUInt32BE(dataLen, 0);
    chunk.write('tEXt', 4);
    keyBuffer.copy(chunk, 8);
    chunk[8 + keyBuffer.length] = 0; // Null separator
    textBuffer.copy(chunk, 8 + keyBuffer.length + 1);

## 5. 검증 계획
1.  이미지 생성 후 저장.
2.  저장된 이미지 파일을 텍스트 에디터 등으로 열어 `tEXt` 청크와 프롬프트가 포함되어 있는지 확인 (또는 사이드바에서 확인).
3.  AI Studio 사이드바에서 '프롬프트 복사' 버튼 클릭 시 원본 프롬프트가 복사되는지 확인.
