```typescript
const ShortsImageHistorySidebar: React.FC<Props> = ({
  show,
  favoritesOnly,
  setFavoritesOnly,
  onClose,
  imageHistory,
  historyUrls,
  onToggleFavorite,
  onCopyPrompt,
  onDelete,
  onSelectImage,
  enableDrag = true,
  onRefresh,
  currentStoryId  // ✅ 추가
}) => {
  // ✅ 필터링 로직 추가
  const filteredHistory = useMemo(() => {
    if (!currentStoryId) {
      // currentStoryId가 없으면 전체 이미지 표시
      return imageHistory;
    }
    
    // currentStoryId와 일치하는 이미지만 필터링
    const filtered = imageHistory.filter(item => 
      item.storyId === currentStoryId
    );
    
    // 필터링된 이미지가 없으면 전체 이미지 표시
    return filtered.length > 0 ? filtered : imageHistory;
  }, [imageHistory, currentStoryId]);

  // orderedHistory를 filteredHistory 기반으로 변경
  const orderedHistory = useMemo(() => {
    const sorted = [...filteredHistory].sort((a, b) => {
      const favDiff = Number(Boolean(b?.favorite)) - Number(Boolean(a?.favorite));
      if (favDiff !== 0) return favDiff;
      const timeDiff = (b?.createdAt ?? 0) - (a?.createdAt ?? 0);
      if (timeDiff !== 0) return timeDiff;
      return 0;
    });
    return favoritesOnly ? sorted.filter(item => item.favorite) : sorted;
  }, [filteredHistory, favoritesOnly]);  // ✅ imageHistory 대신 filteredHistory 사용
  
  // ... 나머지 코드
};
```

## 🧪 **테스트 계획**

### 테스트 시나리오

#### 시나리오 1: 테스트 대본에 이미지가 있는 경우
1. **준비**: "골프장의_위험한_재회-3" 테스트 대본 생성, 이미지 2개 생성
2. **실행**: 보관함 → 테스트 대본 → 해당 대본 클릭
3. **예상 결과**: 
   - 사이드바에 해당 대본의 이미지 2개만 표시
   - 헤더에 "📁 골프장의_위험한_재회-3" 표시

#### 시나리오 2: 테스트 대본에 이미지가 없는 경우
1. **준비**: "새로운_테스트_대본" 생성 (이미지 없음)
2. **실행**: 보관함 → 테스트 대본 → 해당 대본 클릭
3. **예상 결과**:
   - 사이드바에 전체 이미지 히스토리 표시
   - 필터 헤더는 표시되지 않음

#### 시나리오 3: 대본 전환
1. **준비**: 테스트 대본 A (이미지 3개), 테스트 대본 B (이미지 2개)
2. **실행**: 
   - 대본 A 클릭 → 이미지 3개 확인
   - 대본 B 클릭 → 이미지 2개 확인
3. **예상 결과**: 각 대본의 이미지만 정확히 표시

## ⚠️ **주의사항**

1. **"생성 대본"에는 이미지 사이드바가 없음**
   - 필요하다면 나중에 추가 가능
   - 현재는 "테스트 대본"만 지원

2. **기존 이미지 호환성**
   - `storyId`가 없는 이미지는 전체 표시에 포함
   - `orphaned` 폴더 이미지도 전체 표시에 포함

3. **성능**
   - `useMemo`로 최적화되어 있음
   - `imageHistory`가 커도 문제없음
