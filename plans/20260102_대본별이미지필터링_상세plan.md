**수정 후**:
```typescript
  // ✅ 추가: currentStoryId로 필터링
  const filteredHistory = useMemo(() => {
    if (!currentStoryId) {
      // currentStoryId가 없으면 전체 이미지 표시
      return imageHistory;
    }
    
    // currentStoryId와 일치하는 이미지만 필터링
    const filtered = imageHistory.filter(item =>

    // 필터링된 이미지가 없으면 전체 이미지 표시 (폴백)
    return filtered.length > 0 ? filtered : imageHistory;
  }, [imageHistory, currentStoryId]);

  // ✅ 수정: imageHistory 대신 filteredHistory 사용
  const orderedHistory = useMemo(() => {
    const sorted = [...filteredHistory].sort((a, b) => {
      const favDiff = Number(Boolean(b?.favorite)) - Number(Boolean(a?.favorite));
      if (favDiff !== 0) return favDiff;
      const timeDiff = (b?.createdAt ?? 0) - (a?.createdAt ?? 0);
      if (timeDiff !== 0) return timeDiff;

```


```
3. 이미지가 생성될 때까지 대기

```

### 테스트 시나리오 1: 대본에 이미지가 있는 경우

**목표**: 해당 대본의 이미지만 필터링되어 표시되는지 확인

**단계**:
1. "테스트1" 대본에 이미지 2개 생성
2. "테스트2" 대본 생성 후 이미지 1개 생성
3. 보관함 → 테스트 대본 → "테스트1" 클릭
4. 오른쪽 사이드바 확인

**예상 결과**:
- ✅ 사이드바 헤더에 "📁 테스트1" 표시
- ✅ "테스트1"의 이미지 2개만 표시
- ✅ "테스트2"의 이미지는 보이지 않음

**확인 방법**:
```
사이드바 헤더를 보고 폴더명 확인
이미지 개수가 2개인지 확인
```

### 테스트 시나리오 2: 대본에 이미지가 없는 경우

**목표**: 전체 이미지가 표시되는지 확인 (폴백 동작)

**단계**:

2. 보관함 → 테스트 대본 → "테스트3" 클릭
3. 오른쪽 사이드바 확인

**예상 결과**:
- ✅ 사이드바 헤더에 폴더명 표시 없음
- ✅ 전체 이미지 히스토리 표시 (다른 대본의 이미지 포함)

**확인 방법**:
```
사이드바 헤더에 "📁 ..." 텍스트가 없는지 확인

```

## 🐛 디버깅 가이드

### 문제 1: 필터링이 작동하지 않음

**증상**: 모든 이미지가 계속 표시됨

**확인 사항**:
1. `currentStoryId`가 제대로 전달되는지 확인
   ```typescript
   // youtube-shorts-script-generator.tsx 3151번 라인 다음에 추가
   console.log('[DEBUG] currentStoryId:', (viewingStory as any)?._folderName);
   ```

2. `imageHistory`의 `storyId` 값 확인
   ```typescript
   // ShortsImageHistorySidebar.tsx filteredHistory 내부에 추가
   console.log('[DEBUG] Filtering:', {
     currentStoryId,
     imageHistory: imageHistory.map(item => ({ id: item.id, storyId: item.storyId }))
   });
   ```

**해결 방법**:
- `viewingStory._folderName`이 `undefined`인 경우: 대본 생성 시 `_folderName`이 설정되지 않음 → 이전 커밋 확인
- `imageHistory[].storyId`가 `undefined`인 경우: 이미지 생성 시 `storyId`가 설정되지 않음 → 이전 커밋 확인

### 문제 2: 사이드바 헤더가 표시되지 않음

**증상**: 필터링은 작동하지만 "📁 폴더명"이 보이지 않음

**확인 사항**:
1. `currentStoryId`가 `truthy` 값인지 확인

**해결 방법**:
- `currentStoryId`가 빈 문자열인 경우: `viewingStory._folderName`이 빈 문자열 → 서버 로직 확인
- 조건부 렌더링 `{currentStoryId && ...}`이 올바른지 확인

---

### 문제 3: 대본 전환 시 필터링이 업데이트되지 않음

**증상**: 첫 번째 대본의 이미지가 계속 표시됨

**확인 사항**:
1. `viewingStory`가 업데이트되는지 확인


2. `useMemo` 의존성 배열 확인
   ```typescript
   // ShortsImageHistorySidebar.tsx filteredHistory의 의존성 배열 확인
   }, [imageHistory, currentStoryId]);  // ✅ currentStoryId가 포함되어 있는지 확인
   ```

**해결 방법**:
- `useEffect`가 실행되지 않는 경우: `selectedStory` prop이 변경되지 않음 → `App.tsx` 확인
- `useMemo`가 재계산되지 않는 경우: 의존성 배열에 `currentStoryId` 추가
