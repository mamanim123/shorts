# Implementation Plan - 폴더 목록 최신순 정렬

## 1. 개요
- **작업 목표**: AI Studio 우측 사이드바의 폴더 드롭다운 메뉴가 최신순(최근 수정된 순서)으로 정렬되도록 수정한다.
- **요청자**: 마마님
- **작성일**: 2026-01-02

## 2. 현상 분석
- 현재 `server/index.js`의 `listImageStoryFolders` 함수는 `fs.readdirSync`를 사용하여 폴더 목록을 가져온 후, 별도의 정렬 과정 없이 반환하고 있다.
- 이로 인해 운영체제의 파일 시스템 정렬 기준(보통 이름순 또는 생성순)에 따라 폴더가 표시되고 있어, 사용자가 최근 작업한 폴더를 찾기 어렵다.

## 3. 수정 계획
### 대상 파일
- `server/index.js`

### 수정 내용
- `listImageStoryFolders` 함수 내부 로직 변경
    1. `fs.readdirSync`로 폴더 목록을 가져온다.
    2. 각 폴더에 대해 `fs.statSync`를 호출하여 `mtimeMs` (마지막 수정 시간) 정보를 가져온다.
    3. `mtimeMs`를 기준으로 **내림차순(최신순)** 정렬한다.
    4. 정렬된 순서대로 기존 로직(이미지 개수 카운트 등)을 수행하여 반환한다.

### 코드 변경 예시
```javascript
const listImageStoryFolders = () => {
    if (!fs.existsSync(IMAGES_DIR)) {
        return [];
    }
    return fs.readdirSync(IMAGES_DIR)
        .map(entry => {
            try {
                const fullPath = path.join(IMAGES_DIR, entry);
                const stat = fs.statSync(fullPath);
                return {
                    folderName: entry,
                    isDirectory: stat.isDirectory(),
                    mtimeMs: stat.mtimeMs
                };
            } catch {
                return null;
            }
        })
        .filter(item => item && item.isDirectory)
        .sort((a, b) => b.mtimeMs - a.mtimeMs) // 최신순 정렬
        .map(({ folderName }) => ({
            folderName,
            imageCount: (() => {
                try {
                    return fs.readdirSync(path.join(IMAGES_DIR, folderName))
                        .filter(file => /(png|jpg|jpeg)$/i.test(file))
                        .length;
                } catch {
                    return 0;
                }
            })()
        }));
};
```

## 4. 검증 계획
- 수정 후 서버를 재시작한다.
- AI Studio에 접속하여 우측 사이드바의 폴더 드롭다운 메뉴를 확인한다.
- 가장 최근에 이미지를 생성하거나 수정한 폴더가 목록의 최상단에 위치하는지 확인한다.
