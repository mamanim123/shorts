if (!includesGroupShotTag(scene.longPrompt, scene.characterIds)) {
  violations.push(
    `스크립트 ${scriptIndex + 1} Scene ${scene.sceneNumber}: 다중 등장 인물에 Two-shot/Three-shot 태그가 없습니다.`

**includesGroupShotTag() 함수 (라인 58-63)**:
```typescript
const includesGroupShotTag = (prompt?: string, characterIds?: string[]) => {
  if (!prompt) return false;
  if (!requiresGroupShot(characterIds)) return true;  // 1명이면 패스
  const normalized = prompt.toLowerCase();
  return normalized.includes('two-shot') || normalized.includes('three-shot');
};
```

## 🔍 3. Scene 7 오류 상세 원인

### 문제 상황
- AI가 Scene 7의 longPrompt를 생성할 때
- **characterIds에는 2명 이상 포함됨** (A, B 등)
- 하지만 **longPrompt 텍스트에 'two-shot' 또는 'three-shot' 단어가 없음**

### 왜 발생했는가?

#### 템플릿 명시
```
**🚨🚨🚨 CRITICAL WARNING 🚨🚨🚨**
**3. 2명 이상 등장하는 씬은 반드시 프롬프트 맨 앞에 "Two-shot" 또는 "Three-shot" 구도를 명시하세요!**
```

#### AI 응답
```
Scene 7. Slot A — narrator, Slot B — friend, 40s Korean women, ...
(Two-shot 없음!)
```

### 근본 원인
1. **템플릿 지침은 매우 명확함**
2. **AI가 프롬프트를 받았지만 실제 생성 시 이를 따르지 않음**
3. **가능한 이유**:
   - AI가 characterIds와 longPrompt를 독립적으로 생성하여 불일치
   - 프롬프트 토큰 길이 제약으로 인한 생략
   - 장르 가이드(genrePrompt)와 scriptImage 템플릿의 우선순위 충돌

## 🔍 5. 프롬프트 전달 체인 분석

### 파일: `services/promptBuilder.ts:31-147`

### 최종 프롬프트 조합 과정

```
1. Mode Template 로드 (scriptImage)
   ↓
2. 변수 바꾸기 ({{GENRE}}, {{TOPIC}}, {{OUTFIT_BLOCK}} 등)
   ↓
3. 장르 가이드 추가 (genreGuideline.prompt)
   ↓
4. 최종 프롬프트 조합
   parts = [
     genrePrompt,          ← 장르 가이드 (먼저)
     filledTemplate        ← scriptImage 템플릿 (나중)
   ]
   ↓
5. 유니크 토큰 추가 (uniqueFooter)
   ↓
6. AI에 전달
```

### 발견된 문제점

#### 문제 1: 장르 가이드 우선순위

**코드 (라인 134-142)**:
```typescript
const parts = [
  genrePrompt,              // 장르 가이드가 먼저!
  filledTemplate && `[생성 모드 템플릿]\n${filledTemplate}`  // scriptImage는 나중
].filter(Boolean);

const fullPrompt = parts.join('\n\n');
```

**문제**:
- 장르 가이드(genrePrompt)가 scriptImage 템플릿보다 **먼저** 조합됨
- 장르 가이드가 자체 프롬프트 구조를 제공하면, scriptImage의 세부 지침(Two-shot, tight-fitting 등)이 **약화되거나 후순위**로 처리될 가능성
- AI가 토큰 제한이나 우선순위에 따라 선택할 때, **나중에 오는 세부 지침을 생략**할 수 있음

#### 문제 2: 유니크 토큰의 주의 분산

**코드 (라인 144-146)**:
```typescript
const uniqueFooter = `\n\n${'='.repeat(80)}\n[Unique Request ID: {{CREATIVITY_BOOSTER}}]\n[Timestamp: ${new Date().toISOString()}]`;
return fullPrompt + uniqueFooter;
```

**문제**:
- 유니크 토큰이 프롬프트 맨 끝에 위치
- AI의 **집중력을 분산**시킬 수 있음
- AI가 유니크 토큰 처리에 집중하느라 세부 지침(Two-shot, tight-fitting) 놓칠 가능성

## 🎯 7. 해결 방향 제안 (수정 금지)

### 제안 1: 프롬프트 계층 구조 명확화

**현재**:
```
[장르 가이드] + [scriptImage 템플릿]
```

**개선**:
```
[필수 지침 요약 (Two-shot, tight-fitting 등)]
  ↓
[장르 가이드]
  ↓
[scriptImage 상세 템플릿]
  ↓
[유니크 토큰]
```

### 제안 2: 필수 태그 강조도 증가

**현재**:
```
**[필수 6] 의상 스타일 강조 (절대 생략 금지!)**
```

**개선**:
```
**[MANDATORY - NEVER OMIT] Outfit Style Tags:**
- tight-fitting premium tailored design
- bodycon silhouette accentuating feminine curves
- form-fitting elegant attire with body-conscious refinement

**⚠️ If you omit any of these, the response will be rejected!**
```

### 제안 3: CharacterIds와 프롬프트 동기화

**현재**:
- characterIds와 longPrompt를 AI가 독립적으로 생성

**개선**:
- characterIds.length >= 2이면 **프롬프트 앞에 "Two-shot 필수"를 사전 삽입**
- AI가 선택할 여지 제거

### 제안 4: 검증 로직 완화 (선택적)

**현재**:
- REQUIRED_FIT_TAGS 3개 모두 필수

**개선 옵션**:
- 3개 중 **최소 1개만** 포함되면 패스
- 또는 템플릿에 "3개 중 최소 1개 포함" 명시

### 제안 5: 프롬프트 사전 검증

**현재**:
- AI 응답 후 검증 (후처리)

**개선**:
- 프롬프트 생성 과정에서 **중간 체크**
- characterIds 결정 → Two-shot 프롬프트 자동 삽입 확인

## 🎓 최종 결론

### 왜 이런 오류가 발생했는가?

1. **템플릿은 완벽함**: Two-shot, tight-fitting 모두 명확히 명시
2. **검증 로직도 완벽함**: 누락 감지가 정확함
3. **하지만 AI가 따르지 않음**: 프롬프트를 받았지만 일부 지침을 생략

### 근본 원인

**프롬프트 전달 체인에서의 우선순위 문제**:
- 장르 가이드(genrePrompt)가 먼저 전달됨
- scriptImage 템플릿의 세부 지침이 후순위로 처리됨
- AI가 토큰 제약이나 우선순위에 따라 세부 지침을 생략함

### 해결 방향

1. 필수 지침을 프롬프트 **최상단에 요약** 배치
2. 장르 가이드와 scriptImage의 **우선순위 명확화**
3. CharacterIds와 프롬프트 생성의 **동기화**
4. 검증 로직 **완화** 고려 (3개 → 최소 1개)
5. 유니크 토큰 위치 **최적화**
