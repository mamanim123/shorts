## 📋 Executive Summary (요약)

마마님께서 대본+이미지프롬프트+장르 선택 후 생성 시 **의도한 대로 결과가 나오지 않는 문제**의 근본 원인을 발견했습니다.

**5가지 핵심 문제점**:
1. ⚠️ 장르 정보 누락 가능성 (genre='none' 선택 시 완전 손실)
2. ⚠️ 의상 정보 누락 위험 (폴백 부재로 undefined 반환 가능)
3. ⚠️ 필수 문구 검증 부족 (scene 검증이 일부만 수행)
4. ⚠️ 프롬프트 크기 초과 (Random 모드 시 토큰 제한 초과)
5. ⚠️ 장르→프롬프트 매핑 오류 (커스텀 장르 손실)

## 🚨 2. 발견된 핵심 문제점

### 문제 1: 장르 정보 완전 누락 (Critical)

**파일**: `services/promptBuilder.ts:127-129`

**코드**:
```ts
const genrePrompt = genreGuideline && genreGuideline.id !== 'none'
  ? fillTemplate(`[장르: ${genreGuideline.name}]\n${genreGuideline.prompt}`, variables)
  : null;  // ⚠️ 'none' 선택 시 완전 제외
```

**문제점**:
- 사용자가 장르를 'none'으로 선택하거나
- `genre` 파라미터가 undefined일 경우
- **장르 특성이 완전히 손실됨** → AI가 무작위 스타일로 생성

**영향**:
- "샤베트코미디"를 선택했는데 일반 대본이 나옴
- "역사" 장르인데 현대물 톤으로 생성됨

**해결책**:
```ts
// 기본값 설정
const effectiveGenre = genre && genre !== 'none' ? genre : 'affair-suspicion';
const genreGuideline = getGenreGuideline(effectiveGenre);
```

### 문제 2: 의상 정보 누락 (Critical)

**파일**: `youtube-shorts-script-generator.tsx:1846-1876`

**코드**:
```ts
let resolvedFemale = resolveOutfit(effectiveInput)?.trim() || '';

if (!resolvedFemale) {  // ⚠️ 폴백 필요
  resolvedFemale = pickRandomOutfit();  // 실패 시 undefined 가능
}
```

**문제점**:
1. `resolveOutfit()` 함수가 의상을 찾지 못한 경우
2. `pickRandomOutfit()` 배열이 비어있는 경우
3. **undefined가 반환되어 의상 정보가 완전히 빠짐**

**실제 버그 코드**:
```ts
const pickRandom = <T,>(array: T[]): T => {
  if (array.length === 0) return undefined;  // ⚠️ undefined 반환!
  return array[Math.floor(Math.random() * array.length)];
};
```

**영향**:
- 이미지 프롬프트에 "Korean woman" 만 나오고 의상 없음
- AI가 임의로 의상을 생성 → 씬마다 의상이 달라짐
- **일관성 완전 손실**

**해결책**:
```ts
const pickRandom = <T,>(array: T[], fallback?: T): T => {
  if (array.length === 0) {
    if (fallback) return fallback;
    throw new Error('Array is empty and no fallback provided');
  }
  return array[Math.floor(Math.random() * array.length)];
};

// 사용 시
const resolvedFemale = resolveOutfit(effectiveInput)?.trim() ||
                      pickRandomOutfit() ||
                      'White Halter-neck Knit + Red Mini Skirt';  // 하드코딩 기본값
```

### 문제 3: Scene 검증 부족 (High)

**파일**: `services/promptValidator.ts:53-194`

**코드**:
```ts
const requiresGroupShot = (characterIds?: string[]) => {
  if (!Array.isArray(characterIds)) return false;  // ⚠️ 없으면 false
  return characterIds.length >= 2;
};

const includesGroupShotTag = (prompt?: string, characterIds?: string[]) => {
  if (!requiresGroupShot(characterIds)) return true;  // ⚠️ 검증 안 함
  return /Two-shot|Three-shot/i.test(prompt || '');
};
```

**문제점**:
- `characterIds`가 누락되면 **검증을 아예 안 함**
- Scene에 2명이 필요한데 감지 못함
- 필수 품질 태그 (no text, --ar 9:16) 누락 여부 체크 안 함

**영향**:
- "Two-shot" 없는데 2명 대화 씬이 생성됨
- 텍스트가 이미지에 포함될 수 있음
- 종횡비가 잘못됨 (9:16 아님)

**해결책**:
```ts
// characterIds 필수 체크 추가
if (!scene.characterIds || scene.characterIds.length === 0) {
  violations.push(`Scene ${scene.sceneNumber}: characterIds가 빠졌습니다.`);
}

// 필수 품질 태그 체크
const REQUIRED_QUALITY_TAGS = ['no text', 'no letters', '--ar 9:16'];
REQUIRED_QUALITY_TAGS.forEach(tag => {
  if (!scene.longPrompt.includes(tag)) {
    violations.push(`Scene ${scene.sceneNumber}: 필수 태그 "${tag}"가 없습니다.`);
  }
});
```

### 문제 5: 장르→프롬프트 매핑 실패 (Medium)

**파일**: `services/genreGuidelines.ts`

**코드**:
```ts
const getGenreGuideline = (genreId: string) => {
  const genre = genres.find(g => g.id === genreId);
  return genre || null;  // ⚠️ null 반환 → 장르 완전 손실
};
```

**문제 시나리오**:
1. 사용자가 커스텀 장르 생성 (예: "my-custom-genre")
2. 브라우저 localStorage 초기화 또는 다른 PC 사용
3. 다시 프로그램 실행 → 장르 ID가 genreManager에 없음
4. **null 반환 → genrePrompt 완전 제외**

**영향**:
- 커스텀 장르 선택했는데 기본 스타일로 생성
- 일관성 손실

**해결책**:
```ts
const getGenreGuideline = (genreId: string) => {
  if (!genreId || genreId === 'none') return null;
  const genre = genres.find(g => g.id === genreId);
  // 기본값 폴백
  return genre || getGenreGuideline('affair-suspicion');
};
```

## 🎓 8. 결론

### 8.1 핵심 원인
마마님께서 몇 달간 헤매신 문제의 근본 원인은 **5가지 핵심 버그**:

1. **장르 누락** → AI가 무작위 스타일로 생성
2. **의상 누락** → 씬마다 의상이 달라짐
3. **검증 부족** → 필수 태그 없이 생성됨
4. **토큰 초과** → API 호출 실패
5. **장르 매핑 오류** → 커스텀 장르 손실

### 8.2 권장 조치
1. **즉시 수정 3가지** (Critical):
   - 장르 기본값 설정
   - 의상 폴백 강화
   - Scene 검증 강화

2. **빠른 수정 2가지** (High):
   - 프롬프트 크기 최적화
   - 장르 폴백 추가

3. **점진적 개선** (Medium):
   - 에러 로깅
   - 모니터링 시스템

### 8.3 예상 효과
수정 후:
