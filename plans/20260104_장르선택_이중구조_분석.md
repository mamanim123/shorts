### 연동 특징

1. **localStorage 공유**:
   - 키: `'shorts-generator-genre'`

2. **useEffect 동기화**:
   - `defaultSettings.shortsGenre` 변경 감지
   - 쇼츠 생성기 로컬 상태 업데이트
   - 단, 이미 다르면 업데이트 (같으면 스킵)

3. **초기값 로드**:
   - 쇼츠 생성기 컴포넌트 마운트 시

## ⚖️ 5. 우선순위 및 최종 적용

### 실제 적용되는 장르

**쇼츠 생성기 창의 로컬 상태 `genre`**

**이유**:
1. `generateScript()` 함수가 **쇼츠 생성기 로컬 `genre`**를 직접 사용
2. 메인화면 선택은 `defaultSettings.shortsGenre`로 전달되지만, **제안일 뿐**
3. 쇼츠 생성기에서 다시 선택하면 **완전히 덮어씌워짐**

### 우선순위

```
쇼츠 생성기 로컬 선택 > 메인화면 왼쪽 패널 선택
```

**시나리오 예시**:

#### 시나리오 1: 메인화면에서만 선택
```
1. 메인화면 왼쪽 패널에서 "샤베트코미디" 선택
2. localStorage 업데이트: 'affair-suspicion' → 'sherbet-comedy'
3. 쇼츠 생성기 창 열기
4. useEffect 감지 → genre 상태 업데이트: 'sherbet-comedy'
```

```
2. 쇼츠 생성기 창 열기: "샤베트코미디" 표시됨
```

## ⚠️ 6. 잠재적 문제점

### 문제 1: 비동기 업데이트 지연

**상황**:
```tsx
useEffect(() => {
  if (defaultSettings?.shortsGenre && defaultSettings.shortsGenre !== genre) {
    setGenre(defaultSettings.shortsGenre);
  }
}, [defaultSettings?.shortsGenre, genre]);
```

**문제**:
- 메인화면에서 변경해도 **즉시 반영 안 될 수 있음**
- useEffect는 다음 렌더링 사이클에서만 실행
- props 전달 지연 가능

**영향**:
- 메인화면에서 변경 → 바로 생성 버튼 클릭 시
- 이전 장르로 생성될 가능성 (매우 낮음)

### 문제 2: UI 일관성

**상황**:
- 두 곳에 장르 선택이 있음
- 사용자가 혼란스러울 수 있음

**영향**:
- "어느 것을 선택해야 하나?" 혼란
- 메인화면에서 선택했는데 쇼츠 생성기에서 다른 값 표시 가능성 (매우 낮음)
