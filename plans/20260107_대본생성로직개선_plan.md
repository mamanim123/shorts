# 20260107 대본 생성 로직 개선 계획서 (v6.1.1)

## 1. 개요
현재 쇼츠 생성기의 대본 생성 로직이 너무 정형화되어 있고, 주제어(TOPIC)를 기계적으로 반복하여 창의성이 떨어지는 문제를 해결하기 위한 개선 계획입니다. `대본생성수정방안.txt`의 전문가 조언을 바탕으로 바이럴 최적화 구조로 변경합니다.

## 2. 주요 수정 목표
- **구조 혁신**: 3초 안에 승부를 보는 '훅+상황 융합' 구조로 변경 (배경 설명 섹션 삭제).
- **자연스러운 전개**: 주제어(TOPIC)의 강제 반복을 제거하고 자연스러운 변주 허용.
- **창의성 극대화**: AI가 복제할 수 있는 예시 문장을 제거하고 '원칙' 중심으로 지침 수정.
- **행동 중심 묘사**: 감정 단어 대신 시각적인 행동과 대사 위주로 작성 유도.

## 3. 상세 수정 사항

### 3.1 `youtube-shorts-script-generator.tsx` 수정
- **`DEFAULT_MODE_TEMPLATES.scriptOnly` 수정**:
    - [대본 구조]에서 '배경 및 상황 설명' 단계 삭제.
    - 훅(1문장)에 상황을 녹여내도록 지침 변경.
    - 문장 수를 8~15문장으로 유연하게 조정.
    - 반전 위치를 '후반부'로 명시하여 고정된 패턴 탈피.
- **`DEFAULT_MODE_TEMPLATES.scriptImage` 수정**:
    - 위와 동일하게 구조 개선.
    - 이미지 프롬프트 생성 시에도 대본의 변화된 리듬을 반영하도록 지침 최적화.
- **공통 규칙 수정**:
    - "각 섹션마다 TOPIC 1회 이상 사용" 규칙을 "전체 대본에서 자연스럽게 3~5회 노출"로 완화.
    - 예시 문장(예: "인생 꼬일 뻔했어") 등을 모두 삭제하여 AI의 자가 복제 방지.

### 3.2 `engine_config.json` 검토 및 최적화
- 각 엔진(CUSTOM_XXX) 프롬프트 내에 "배경 설명"이나 "특정 구조"를 강제하는 문구가 있는지 확인하고, 발견 시 수정 제안 (이번 단계에서는 템플릿 우선 수정).

## 4. 기대 효과
- 첫 문장에서 바로 사건이 시작되어 시청자 이탈률 감소.
- 매번 다른 패턴의 대본이 생성되어 "그 대본이 그 대본 같다"는 느낌 해소.
- AI의 자유도가 높아져 더 기발한 반전과 상황 설정 가능.
