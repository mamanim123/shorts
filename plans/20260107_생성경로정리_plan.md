## 목표
- 쇼츠 대본 생성 관련 버튼/프롬프트 경로를 하나로 통일해 주제를 창의적으로 해석하고, 슬롯/의상 규칙을 일관되게 적용하도록 만든다.

## 전제
- 기존 `buildFinalPrompt` 로직은 주제 복붙 금지, 의상/슬롯 규칙을 포함하고 있으므로 이를 공통 경로로 삼는다.
- Implementation Plan 승인 후 실제 코드 수정에 착수한다.

## 작업 범위
1. `App.tsx` 좌측 패널 버튼 정리
   - `쇼츠 대본 생성`, `쇼츠 기본`, `쇼츠 실험` 버튼 제거 또는 숨김.
   - `스토리 생성`만 유지하거나, 쇼츠 생성기로 이동하는 새로운 단일 버튼을 만든다.
   - 버튼 삭제 시 관련 상태(`autoGenerateShorts`, `__useSimpleTemplate`, `__useTestTemplate`) 사용처 파악.

2. `ShortsScriptGenerator` 프롬프트 구성 단일화
   - `generateScript`에서 `/새로운대본.txt`, `/테스트대본.txt` 템플릿을 로딩하는 분기 제거.
   - 항상 `buildFinalPrompt`를 사용하도록 하고, 필요 시 UI에서 템플릿을 선택하는 기능을 명시적으로 제공.
   - `ModeTemplateSettings` UI를 다시 노출할지, 기능 자체를 없앨지 결정 후 반영.

3. 의상/슬롯 규칙 적용 일관화
   - `resolveEffectiveOutfits`와 `buildFinalPrompt`가 주제/장르를 반영해 랜덤 의상을 선택하도록 로직 확인 및 개선.
   - Slot 시스템/품질 태그가 모든 씬에 적용되는지 검증하고 누락 시 보완.

4. 프롬프트 미리보기 신뢰성 확보
   - `미리보기` 버튼이 실제로 `generateScript`에 전달되는 최종 문자열을 그대로 노출하도록 구조 조정.
   - 프롬프트를 생성하는 함수가 하나뿐임을 보장해 UI와 실제 호출이 다르지 않게 한다.

5. 회귀 테스트 & 검증
   - `스토리 생성`과 `쇼츠 생성기` 모두에서 동일한 주제 입력 시, 주제 문구가 복붙되지 않고 변주되는지 확인.
   - 장면별 의상/슬롯/태그가 요구사항을 만족하는지 생성 결과로 검증.
   - 필요 시 QA용 테스트 스크립트를 업데이트하거나 새 문서를 작성한다.

## 산출물
- 정리된 버튼/UI 코드 (`App.tsx`, `ShortsScriptGenerator`, 관련 컴포넌트).
- 단일화된 프롬프트 빌더 사용 내역과 테스트 결과 요약.
