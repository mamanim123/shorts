# Implementation Plan: Ctrl+C 강제 종료 버그 수정

## 1. 개요
브라우저에서 텍스트 복사를 위해 `Ctrl+C`를 누를 때, 해당 이벤트가 터미널로 전달되어 프로세스가 종료되는 현상을 방지합니다.

## 2. 분석 결과
- 현상: 브라우저에서 `Ctrl+C` 입력 시 앱이 강제 종료됨.
- 원인: 키보드 이벤트가 브라우저 내에서 캡처되지 않고 터미널 프로세스까지 전달되어 `SIGINT` 신호로 해석됨.
- 해결책: 브라우저 최상단(`App.tsx`)에서 `keydown` 이벤트를 가로채 `Ctrl+C`인 경우 이벤트 전파를 중단(`stopPropagation`)시키고 브라우저 내부에서만 처리되도록 함.

## 3. 작업 내용
- `App.tsx` 파일에 `useEffect`를 사용하여 전역 `keydown` 이벤트 리스너 등록.
- `Ctrl` 또는 `Meta`(Mac) 키와 함께 `C` 키가 눌렸는지 확인.
- 해당 조건 충족 시 `e.stopPropagation()`을 호출하여 외부 유출 차단.
- 브라우저 기본 복사 기능은 유지되도록 `preventDefault()`는 호출하지 않음.

## 4. 검증 계획
- `lsp_diagnostics`를 통한 구문 오류 확인.
- 실제 동작 환경에서 `Ctrl+C` 입력 시 창이 닫히는지 여부 확인 (사용자 확인 필요).
