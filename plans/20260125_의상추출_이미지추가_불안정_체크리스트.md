# 의상추출 이미지추가 불안정 점검 계획

## 목표
- 의상추출 플로우에서 이미지 추가/업로드가 실패하는 원인을 재현 가능하게 분리한다.
- 불안정 구간(파일 선택, 업로드, 응답 대기)을 단계별로 확인하고 개선 포인트를 도출한다.
- 안정화 작업 전/후 체크리스트로 품질을 보증한다.

## 현재 관찰 증상
- Puppeteer에서 `FileChooser` 대기 타임아웃(30s)이 간헐적으로 발생.
- 동일 입력에서도 업로드가 성공/실패가 섞여 재현성이 낮음.

## 점검 범위
- UI 동작: `+` 버튼 클릭 → 파일 선택창 발생 여부
- Puppeteer 동작: 클릭 타겟/포커스/DOM 상태 확인
- 업로드 이후 처리: 업로드 완료 이벤트/응답 대기/JSON 파싱
- 캐시/세션: 브라우저 세션 유지, user_data 상태

## 점검 체크리스트
### 1) UI/DOM 상태 점검
- [ ] `+` 버튼이 실제로 파일 input과 연결되어 있는지 확인
- [ ] 클릭 시 input이 `display:none`/`disabled` 상태인지 확인
- [ ] 업로드 UI가 로딩 중에 버튼이 비활성화되는지 확인
- [ ] 드롭존/클릭 업로드 양쪽 경로 테스트

### 2) Puppeteer 이벤트 점검
- [ ] `page.waitForEvent('filechooser')` 타임아웃 로그 수집
- [ ] 클릭 대상이 정확한지(overlay/hidden 버튼 여부) 확인
- [ ] 클릭 직후 DOM 변화를 스냅샷으로 확보
- [ ] 파일 선택창이 실제로 열렸는지 여부 확인

### 3) 파일 처리 점검
- [ ] 업로드 파일 경로/존재 여부 확인
- [ ] 동일 파일로 반복 업로드 시 성공률 체크
- [ ] 파일 크기/확장자 영향 확인

### 4) 서버/응답 처리 점검
- [ ] 업로드 요청이 서버까지 도달하는지 네트워크 로그 확인
- [ ] 추출 API 응답 시간/실패율 기록
- [ ] 실패 시 프론트 에러 핸들링 확인

### 5) 세션/브라우저 안정성
- [ ] Puppeteer 세션 재사용 여부 확인
- [ ] user_data 디렉토리 충돌/손상 여부 확인
- [ ] 멀티 세션 동시 실행 여부 확인

## 산출물
- 불안정 재현 조건 정리
- 실패 지점(클릭/파일선택/업로드/응답) 구분 로그
- 안정화 수정안 초안
