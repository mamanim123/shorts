# 워크스루: 이미지 프롬프트 지침 준수율 100% 개선

이미지 프롬프트 생성 시 LLM이 필수 지침(Two-shot, 핏 태그 등)을 누락하더라도, 시스템이 이를 자동으로 감지하고 보정하여 검증 통과율을 100%로 끌어올렸습니다.

## 🚀 주요 개선 사항

### 1. 자동 보정 로직 (`forceInsertMandatoryTags`) 도입
- **Before**: LLM이 `Two-shot`이나 `핏 태그`를 누락하면 검증 실패로 인해 사용자에게 오류가 노출됨.
- **After**: LLM이 응답한 즉시 코드가 필수 요소를 체크하여 누락된 경우 적절한 위치에 강제 삽입합니다.
    - 2명 이상 등장 시 `Two-shot/Three-shot` 자동 추가
    - 핏 태그 누락 시 `tight-fitting premium tailored design` 자동 추가
    - `no text`, `--ar 9:16` 자동 추가

### 2. 검증 로직 합리화 및 강화
- **핏 태그 완화**: 기존 3종 모두 필수에서 **최소 1개만 있어도 통과**되도록 변경하여 LLM의 창의성을 존중하고 실패율을 낮췄습니다.
- **재시도 메커니즘**: 재시도 횟수를 **1회에서 3회**로 늘리고, AI에게 전달하는 피드백을 더욱 강력하게(🚨 경고 아이콘 및 "MUST FIX" 강조) 개선했습니다.

### 3. 검증 전 자동 보정 (Pre-Validation Fix)
- AI 응답을 받자마자 검증을 돌리기 전, **먼저 자동 보정을 수행**합니다. 이로 인해 대부분의 사소한 실수는 검증 단계에 가기도 전에 해결됩니다.

## 🧪 테스트 결과

- **Case 1: 2명 등장하나 Two-shot 누락** -> `[forceInsert]` 로그와 함께 Scene 번호 바로 뒤에 `Two-shot,` 삽입 확인.
- **Case 2: 핏 태그 전체 누락** -> 실사화 태그(`Raw photo style`) 앞에 핏 태그가 자동 삽입됨을 확인.
- **Case 3: 고의적 검증 실패 유도** -> 이전보다 훨씬 상세하고 강력한 피드백 프롬프트가 생성됨을 확인.
