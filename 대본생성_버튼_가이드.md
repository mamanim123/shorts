## 3. 처리 파이프라인

### 쇼츠 대본 생성 (전체 검증)
```
사용자 입력
  ↓
buildFinalPrompt() - 프롬프트 생성
  ↓
서버 전송 (보낸대본.txt 자동 저장)
  ↓
LLM 응답 (받은대본.txt 자동 저장)
  ↓
validateScriptScenes() - 3회 재시도 검증
  ↓
extractScenesFromMarkdown() - 씬 추출
  ↓
forceInsertMandatoryTags() - 필수 태그 강제 삽입
  ↓
postProcessScripts() - 후처리
  ↓
enhanceScenePrompt() - 이미지 프롬프트 최적화 (×96회)
  ↓
최종 대본 출력
```

### 쇼츠 기본 / 쇼츠 실험 (빠른 생성)
```
사용자 입력
  ↓
buildFinalPrompt() - 템플릿 기반 프롬프트 생성
  ↓
서버 전송 (보낸대본.txt 자동 저장)
  ↓
LLM 응답 (받은대본.txt 자동 저장)
  ↓
extractScenesFromMarkdown() - 씬 추출
  ↓
normalizeSimpleScripts() 호출
  ↓
forceInsertMandatoryTags() - 필수 태그 강제 삽입 ⭐ NEW
  ↓
enhanceScenePrompt() - 이미지 프롬프트 최적화
  ↓
최종 대본 출력
```

## 6. 권장 사용법

### 일반 사용자
**"쇼츠 대본 생성"** 버튼 권장
- 가장 안정적이고 검증된 대본
- 필수 태그 보장으로 이미지 생성 성공률 높음

### 빠른 테스트가 필요한 경우
**"쇼츠 기본 (고정 룰)"** 버튼 권장
- 검증 과정 생략으로 빠른 생성
- 일관된 품질 보장

### 다양한 패턴 실험이 필요한 경우
**"쇼츠 실험 (다양성 강화)"** 버튼 권장
- 35가지 구조 조합
- 창의적인 대본 생성
- 새로운 아이디어 발굴

### 텍스트 스토리만 필요한 경우
**"스토리 생성"** 버튼 권장
- 이미지 없이 빠른 스토리 생성
- 브레인스토밍용

## 8. 문제 해결

### 이미지 생성 실패 시
1. "쇼츠 대본 생성" 모드 사용 (검증 강화)
2. 받은대본.txt 확인하여 필수 태그 누락 여부 체크
3. tight-fitting, Two-shot 등 태그 수동 추가

### 대본이 너무 비슷할 때
1. "쇼츠 실험 (다양성 강화)" 모드 사용
2. 테스트대본.txt의 다양성 규칙 확인
3. 토픽 키워드 변경

### 생성 속도가 너무 느릴 때
1. "쇼츠 기본 (고정 룰)" 모드로 전환
2. 검증 과정 생략으로 4-6초 단축
