# 쇼츠 생성기 보관함 연동 구현 계획

## 목표

쇼츠 생성기 화면의 왼쪽 패널에 저장된 대본 목록을 실시간으로 표시하여, 저장된 대본을 클릭하면 바로 볼 수 있도록 개선합니다.

## 현재 상황

- 쇼츠 생성기는 전체 화면 오버레이로 표시됩니다
- 왼쪽 패널(400px)에는 닫기 버튼과 설명만 있습니다
- 대본을 저장하면 `stories` 배열에 추가되지만, 쇼츠 생성기 화면에서는 확인할 수 없습니다
- 저장하지 않은 대본과 분석 결과는 쇼츠 생성기 오른쪽 패널에 표시됩니다

## 요구사항

1. ✅ 쇼츠 생성기 화면은 그대로 유지
2. ✅ 왼쪽 패널에 저장된 대본 목록 실시간 표시
3. ✅ 저장하지 않은 대본도 오른쪽 패널에서 계속 볼 수 있음
4. ✅ 대본 분석 결과도 오른쪽 패널에서 볼 수 있음
5. ✅ 왼쪽 패널에서 저장된 대본 클릭 시 해당 대본 표시
6. ✅ **저장된 대본은 메인 화면의 보관함에도 저장** (나중에 확인 가능)

### 2. handleSaveTestScript 함수 수정 (되돌리기)

**이전 수정 내용을 되돌립니다:**

```typescript
const handleSaveTestScript = async (scriptData: any) => {
  const scenes: Scene[] = scriptData.scenes && Array.isArray(scriptData.scenes)
    ? scriptData.scenes
    : [];

  const newStory: StoryResponse = {
    id: crypto.randomUUID(),
    createdAt: Date.now(),
    title: scriptData.title,
    titleOptions: [scriptData.title],
    scriptBody: scriptData.script,
    punchline: scriptData.twist,
    scenes: scenes,
    service: 'TEST',
    isFavorite: false
  };

  try {
    const updatedStories = [newStory, ...stories];
    setStories(updatedStories);
    await saveHistory(updatedStories);
    
    // ✅ 자동 선택만 추가 (화면 전환 없음)
    setSelectedStoryId(newStory.id);
    
    alert('보관함에 저장되었습니다!\n\n왼쪽 패널에서 확인하세요.');
  } catch (e) {
    console.error(e);
    alert('저장에 실패했습니다.');
  }
};
```

## 작동 방식

1. 쇼츠 생성기를 열면 왼쪽 패널에 저장된 대본 목록이 표시됩니다
2. 오른쪽 패널에서 대본을 생성하고 분석할 수 있습니다
3. "보관함에 저장" 버튼을 누르면:
   - 대본이 `stories` 배열에 추가됩니다
   - 왼쪽 패널의 목록이 자동으로 업데이트됩니다
   - 저장된 대본이 자동으로 선택됩니다
4. 왼쪽 패널에서 저장된 대본을 클릭하면 해당 대본이 메인 화면의 오른쪽 패널에 표시됩니다
5. 쇼츠 생성기를 닫지 않는 한, 생성한 대본과 분석 결과는 계속 유지됩니다

## 장점

- ✅ 쇼츠 생성기 화면에서 모든 작업 가능
- ✅ 저장하지 않은 대본과 분석 결과 유지
- ✅ 저장된 대본을 즉시 확인 가능
- ✅ 화면 전환 없이 작업 가능
- ✅ 기존 메인 화면의 보관함 기능과 독립적
