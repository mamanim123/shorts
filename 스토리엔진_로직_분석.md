### **2단계: 엔진 프롬프트 로드** (youtube-shorts-script-generator.tsx 라인 1104-1138)

```javascript
// 1. engine_config.json에서 프롬프트 로드 시도
const { prompts } = await ensureEngineConfigLoaded();

// 2. 선택된 엔진의 프롬프트 가져오기
if (targetEngineId !== 'NONE') {
  const candidate = resolvedPrompts[targetEngineId] ?? getEnginePrompt(targetEngineId);
  
  // 3. 프롬프트가 없으면 에러
  if (!candidate || !candidate.trim()) {
    showToast("엔진 프롬프트를 찾을 수 없습니다", 'error');
    return; // 여기서 중단!
  }
  
  // 4. {{TARGET_AGE}} 같은 변수 치환
  engineSystemPrompt = candidate
    .replace(/{{TARGET_AGE}}/g, targetLabel)
    .replace(/{{TARGET_AGE_EN}}/g, targetAgeEn);
}
```

**⚠️ 현재 문제점:**
- `ensureEngineConfigLoaded()`는 **프론트엔드에서만** `engine_config.json`을 로드
- 하지만 실제 AI 요청은 **서버**에서 처리
- **서버는 이 엔진 프롬프트를 전혀 모름!**

app.post('/api/generate/raw', async (req, res) => {
  const { service, prompt, promptPresetId } = req.body;

## 🔴 **현재 시스템의 문제점 정리**

### **문제 1: 엔진 프롬프트가 실제로 적용되지 않음**

**현재 흐름:**
```
프론트엔드: engine_config.json 로드 ✅
           ↓
프론트엔드: 엔진 프롬프트 + 사용자 프롬프트 조합 ✅
           ↓
서버: 조합된 프롬프트 그대로 AI에 전달 ✅
```

**하지만!** 
- 프론트엔드에서 `ensureEngineConfigLoaded()`를 호출해도
- **서버가 실제 AI 요청을 처리할 때는 이 정보를 모름**
- 결과적으로 **엔진 프롬프트가 제대로 적용 안 됨**

### **문제 2: 서버가 engine_config.json을 읽지 않음**

서버 코드를 보면:
```javascript
// server/index.js에는 engine_config.json 로드 로직이 없음!
app.post('/api/generate/raw', async (req, res) => {
  const { service, prompt } = req.body;
  // engineVersion을 받지도 않고
  // engine_config.json을 읽지도 않음
  const responseText = await generateContent(service, prompt);
});
```

### **문제 3: 프론트엔드와 서버 간 정보 불일치**

- 프론트엔드: `engineVersion`을 알고 있음
- 서버: `engineVersion`을 전혀 모름
- 결과: 엔진 선택이 무의미해짐

## ✅ **해결 방법**

### **방법 1: 서버에서 엔진 프롬프트 처리 (권장)**

**수정 파일: `server/index.js`**

```javascript
app.post('/api/generate/raw', async (req, res) => {
  const { service, prompt, engineVersion } = req.body; // engineVersion 추가
  
  // engine_config.json 로드
  const configPath = path.join(__dirname, '../engine_config.json');
  const engineConfig = JSON.parse(fs.readFileSync(configPath, 'utf8'));
  
  // 엔진 프롬프트 가져오기
  let finalPrompt = prompt;
  if (engineVersion && engineVersion !== 'NONE') {
    const enginePrompt = engineConfig.prompts[engineVersion];
    if (enginePrompt) {
      // 변수 치환
      const targetAge = req.body.targetAge || '40대';
      const processedEnginePrompt = enginePrompt
        .replace(/{{TARGET_AGE}}/g, targetAge)
        .replace(/{{TARGET_AGE_EN}}/g, targetAge.replace('대', 's'));
      
      finalPrompt = `${processedEnginePrompt}\n\n${prompt}`;
      console.log(`[Server] Applied engine prompt: ${engineVersion}`);
    } else {
      console.warn(`[Server] Engine prompt not found: ${engineVersion}`);
    }
  }
  
  const responseText = await generateContent(service, finalPrompt);
  res.json({ rawResponse: responseText });
});
```

### **방법 2: 프론트엔드 수정**

**수정 파일: `youtube-shorts-script-generator.tsx`**

```javascript
// API 호출 시 engineVersion과 targetAge 전달
body: JSON.stringify({
  service: targetService,
  prompt: promptBody,  // 엔진 프롬프트 제외한 사용자 프롬프트만
  engineVersion: targetEngineId,  // 엔진 버전 추가
  targetAge: targetLabel  // 타겟 연령 추가
})
```

## 📊 수정 전후 비교

### **수정 전:**
```
프론트엔드: 엔진 프롬프트 로드 → 조합 → 서버 전송
서버: 받은 프롬프트 그대로 AI 전송
문제: 서버가 엔진 정보를 모름
```

### **수정 후:**
```
프론트엔드: 사용자 프롬프트 + engineVersion → 서버 전송
서버: engineVersion으로 engine_config.json 읽기 → 조합 → AI 전송
해결: 서버가 엔진 프롬프트를 정확히 적용
```

## 🎯 결론

**핵심 문제:**
- 프론트엔드에서 엔진 프롬프트를 조합하지만
- 서버는 이미 조합된 프롬프트만 받아서
- 엔진 선택이 제대로 반영되지 않음

**해결책:**
- 서버에서 `engineVersion`을 받아서
- `engine_config.json`을 직접 읽고
- 프롬프트를 조합하도록 수정

**우선순위:**
1. 서버 수정 (engine_config.json 로드 로직 추가)
2. 프론트엔드 수정 (engineVersion 전달)
3. 테스트 및 검증
