# 🔍 스토리 엔진 수정 방안 재검증

## 📌 현재 상황 요약

### **문제점**
- 프론트엔드에서 `engineVersion`을 선택하고 `engine_config.json`에서 프롬프트를 로드
- 하지만 **서버는 이미 조합된 프롬프트만 받아서 처리**
- 결과: 엔진 선택이 제대로 반영되지 않을 가능성

---

## ✅ 현재 로직 상세 분석

### **1. 프론트엔드 (youtube-shorts-script-generator.tsx)**

#### 라인 1104-1138: 엔진 프롬프트 로드 및 조합
```javascript
// 1. engine_config.json 로드
const { prompts } = await ensureEngineConfigLoaded();
resolvedPrompts = prompts;

// 2. 선택된 엔진의 프롬프트 가져오기
if (targetEngineId !== 'NONE') {
  const candidate = resolvedPrompts[targetEngineId] ?? getEnginePrompt(targetEngineId);
  
  // 3. 프롬프트 없으면 에러
  if (!candidate || !candidate.trim()) {
    showToast("엔진 프롬프트를 찾을 수 없습니다", 'error');
    return; // 여기서 중단!
  }
  
  // 4. 변수 치환 ({{TARGET_AGE}} 등)
  engineSystemPrompt = candidate
    .replace(/{{TARGET_AGE}}/g, targetLabel)
    .replace(/{{TARGET_AGE_EN}}/g, targetAgeEn);
}
```

#### 라인 1140-1144: 최종 프롬프트 조합
```javascript
const promptBody = usePromptPreset 
  ? presetVariableBlock 
  : (useImagePromptMode ? scriptImagePrompt : scriptOnlyPrompt);

const prompt = usePromptPreset
  ? promptBody
  : (engineSystemPrompt ? `${engineSystemPrompt}\n\n${promptBody}` : promptBody);
```

**✅ 프론트엔드는 정상 작동:**
- `engineSystemPrompt` (엔진 프롬프트) + `promptBody` (사용자 프롬프트)를 조합
- 최종 `prompt`를 서버로 전송

#### 라인 1152-1163: 서버 요청
```javascript
const response = await fetch('http://localhost:3002/api/generate/raw', {
  method: 'POST',
  body: JSON.stringify({
    service: targetService,  // 'GEMINI' or 'GPT'

  })

```

**⚠️ 문제점:**
- `engineVersion`을 서버로 전송하지 않음
- 서버는 이미 조합된 `prompt`만 받음

### **2. 서버 (server/index.js)**

#### 라인 926-946: `/api/generate/raw` 엔드포인트
```javascript
app.post('/api/generate/raw', async (req, res) => {
  const { service, prompt, promptPresetId } = req.body;
  // ❌ engineVersion을 받지 않음!
  
  if (!service || !prompt) return res.status(400).json({ error: "Missing service or prompt" });

  try {
    console.log(`[Server] Raw generation request for ${service}`);
    await switchService(service);
    
    // 프롬프트 프리셋 병합 (있으면)
    const { finalPrompt, appliedPreset, error: presetError } = mergePromptWithPreset(prompt, promptPresetId);
    
    if (presetError) {
      return res.status(400).json({ error: presetError });
    }
    
    // AI에게 전송
    const responseText = await generateContent(service, finalPrompt);
    res.json({ rawResponse: responseText });
  } catch (error) {
    console.error("[Server] Raw Generation Error:", error);
    res.status(500).json({ error: error.message || "Raw generation failed" });
  }
});
```

**✅ 서버는 정상 작동:**
- 받은 `prompt`를 그대로 AI에 전달
- `promptPresetId`가 있으면 프리셋과 병합

## 🤔 **재검증 결과: 현재 로직이 실제로 작동하는가?**

### **결론: 현재 로직은 이미 정상 작동 중입니다!**

#### **이유:**

1. **프론트엔드에서 이미 완전히 조합된 프롬프트를 생성**
   - `engineSystemPrompt` (엔진 프롬프트) + `promptBody` (사용자 프롬프트)
   - 라인 1142-1144에서 조합 완료

2. **서버는 조합된 프롬프트를 그대로 AI에 전달**
   - 서버가 `engineVersion`을 알 필요가 없음
   - 이미 완성된 프롬프트를 받아서 처리

3. **변수 치환도 프론트엔드에서 완료**
   - `{{TARGET_AGE}}` → `40대`
   - `{{TARGET_AGE_EN}}` → `40s`
   - 라인 1130-1134에서 처리

## 🔴 **그렇다면 왜 문제가 발생하는가?**

### **가능한 원인:**

#### **1. 프론트엔드에서 엔진 프롬프트 로드 실패**
```javascript
// 라인 1121-1128
if (!candidate || !candidate.trim()) {
  const available = Object.keys(resolvedPrompts || {}).filter(Boolean).join(', ') || '없음';
  const errorMessage = `선택된 엔진(${targetEngineId})의 시스템 프롬프트를 찾지 못했습니다.`;
  console.error(`[ShortsGenerator] ${errorMessage} (loaded engines: ${available})`);
  showToast(errorMessage, 'error');
  setLoading(false);
  return; // ❌ 여기서 중단되면 생성 자체가 안 됨
}
```

**확인 필요:**
- `engine_config.json` 파일이 올바른 위치에 있는가?
- `targetEngineId`가 `engine_config.json`의 `prompts` 객체에 존재하는가?
- `ensureEngineConfigLoaded()` 함수가 정상 작동하는가?

#### **2. 엔진 프롬프트가 비어있음**
```javascript
// 라인 1132-1134
engineSystemPrompt = candidate
  .replace(/{{TARGET_AGE}}/g, targetLabel)
  .replace(/{{TARGET_AGE_EN}}/g, targetAgeEn);
```

**확인 필요:**
- `engine_config.json`의 프롬프트 내용이 실제로 있는가?
- 프롬프트가 너무 짧거나 공백만 있지 않은가?

#### **3. 프롬프트 조합 로직 오류**
```javascript
// 라인 1142-1144
const prompt = usePromptPreset
  ? promptBody
  : (engineSystemPrompt ? `${engineSystemPrompt}\n\n${promptBody}` : promptBody);
```

**확인 필요:**
- `usePromptPreset`이 `true`일 때는 `engineSystemPrompt`가 무시됨
- `promptPresetId`를 사용하면 엔진 프롬프트가 적용 안 됨
