# 테스트 버튼 대본 출력 로직 차이점 분석

## 🎯 핵심 차이점 발견

테스트 버튼을 눌렀을 때 **LLM이 생성한 대본을 가져오지 못하는 문제**의 원인을 찾았습니다.

---

## 📍 문제 위치: `youtube-shorts-script-generator.tsx`

### v3.1.2 (정상 작동) - 라인 1733-1754

```typescript
const savedScripts = await Promise.all(processedScripts.map(async (script: any, idx: number) => {
  const scriptTitle = script.title || `Script ${idx + 1}`;
  const payload = {
    title: scriptTitle,
    content: formatScriptContent(script),
    service: targetService,
    folderName: script._folderName
  };

  try {
    const saveResult = await saveStoryFile(payload);  // ✅ 파일 저장 성공
    return {
      ...script,
      _folderName: saveResult.folderName || script._folderName
    };
  } catch (saveError) {
    console.warn('Failed to save script file:', saveError);
    return script;  // ✅ 실패해도 script 반환
  }
}));

setScripts(savedScripts);  // ✅ 저장된 대본을 상태에 설정
```

**동작:**
1. LLM이 생성한 `processedScripts`를 받음
2. 각 대본을 순회하며 `saveStoryFile()`로 서버에 저장
3. 저장 성공 여부와 관계없이 **모든 대본 데이터를 반환**
4. `setScripts(savedScripts)`로 UI에 출력

---

### v3.2.0 (문제 발생) - 라인 1828

```typescript
setScripts(processedScripts);  // ❌ 저장 없이 바로 설정
```

**동작:**
1. LLM이 생성한 `processedScripts`를 받음
2. **서버에 저장하는 과정이 완전히 제거됨**
3. `setScripts(processedScripts)`로 바로 UI에 출력

---

## 🔍 왜 v3.2.0에서 대본을 가져오지 못하는가?

### 가능한 원인 1: 서버 저장 로직 누락으로 인한 폴더 생성 실패

v3.1.2에서는:
- `saveStoryFile()` 호출 시 서버가 폴더를 생성하고 `folderName`을 반환
- 이 `folderName`이 대본 객체에 저장됨
- 이후 이미지 생성 등에 이 폴더를 사용

v3.2.0에서는:
- 서버 저장 과정이 없어서 폴더가 생성되지 않음
- `_folderName`이 정의되지 않을 수 있음
- 이후 프로세스에서 폴더를 찾지 못해 실패

### 가능한 원인 2: `onSave` 콜백 차이

v3.1.2 (라인 1764-1768):
```typescript
if (onSave) {
  savedScripts.forEach((script: any, idx: number) => {
    onSave({ ...script, version: script.version ?? idx + 1 });
  });
}
```

v3.2.0 (라인 1838-1842):
```typescript
if (onSave) {
  processedScripts.forEach((script: any, idx: number) => {
    onSave({ ...script, version: script.version ?? idx + 1 });
  });
}
```

둘 다 `onSave` 호출은 동일하지만, v3.1.2는 **서버에 저장 후 반환된 데이터**를 전달하고, v3.2.0는 **서버에 저장하지 않은 원본 데이터**를 전달합니다.

## 🔧 해결 방법

### v3.1.2의 저장 로직을 v3.2.0에 복원

**파일:** `f:\projact\쇼츠대본생성기-v3.2.0\youtube-shorts-script-generator.tsx`  
**위치:** 라인 1828 이전에 삽입

```typescript

```

**변경 후:**
```typescript
const savedScripts = await Promise.all(processedScripts.map(async (script: any, idx: number) => {
  const scriptTitle = script.title || `Script ${idx + 1}`;
  
  // formatScriptContent 함수가 필요 (v3.2.0에 없다면 v3.1.2에서 복사 필요)
  const payload = {
    title: scriptTitle,

    service: targetService,
    folderName: script._folderName

  try {
    // saveStoryFile import 추가 필요: import { saveStoryFile } from './services/storyService';

      _folderName: saveResult.folderName || script._folderName

  } catch (saveError) {

  }

setScripts(savedScripts);
```

**그리고 `onSave` 호출 부분도 수정:**
```typescript
if (onSave) {

```

## 📋 필요한 추가 작업

```bash
copy "f:\projact\쇼츠대본생성기-v3.1.2\services\storyService.ts" "f:\projact\쇼츠대본생성기-v3.2.0\services\storyService.ts"
```

### 2. `youtube-shorts-script-generator.tsx` 상단에 import 추가
```typescript
import { saveStoryFile } from './services/storyService';
```

### 3. `formatScriptContent` 함수 확인

v3.2.0에 `formatScriptContent` 함수가 있는지 확인 필요. 없다면 v3.1.2에서 복사:

```typescript
const formatScriptContent = (script: any) => {
  const title = script?.title || 'Untitled';
  const sceneLines = Array.isArray(script?.scenes)
    ? script.scenes.map((scene: any, idx: number) => {
      const sceneNumber = scene.sceneNumber || idx + 1;
      return `
[Scene ${sceneNumber}]
KR: ${scene.shortPromptKo || ''}
EN: ${scene.shortPrompt || ''}
Long: ${scene.longPrompt || ''}
`;
    }).join('\n')
    : '';

  return `TITLE: ${title}
DATE: ${new Date().toLocaleString()}

=== SCRIPT ===
${script?.script || script?.scriptBody || ''}

=== PUNCHLINE ===
${script?.punchline || script?.twist || script?.hook || ''}

=== SCENES (IMAGE PROMPTS) ===
${sceneLines}
`;
};
```

## ⚡ 추가 차이점: API 엔드포인트

### v3.1.2
```typescript
const response = await fetch('http://localhost:3002/api/generate', {
  // ...
  body: JSON.stringify({
    service: targetService,
    prompt: fullPrompt,
    responseFormat: 'raw'  // ✅ responseFormat 파라미터 사용
  })
});
```

### v3.2.0
```typescript
const response = await fetch('http://localhost:3002/api/generate/raw', { // ❌ 다른 엔드포인트
  // ...
  body: JSON.stringify({
    service: targetService,
    prompt: fullPrompt
    // responseFormat 파라미터 없음
  })
});
```

**차이점:**
- v3.1.2: `/api/generate` + `responseFormat: 'raw'` 파라미터
- v3.2.0: `/api/generate/raw` 엔드포인트 직접 호출

이 차이가 서버 응답 형식에 영향을 줄 수 있습니다.

## 📝 결론

**핵심 문제:**
v3.2.0에서 **서버 저장 로직이 완전히 제거**되어:
1. 대본 파일이 디스크에 저장되지 않음
2. 폴더가 생성되지 않아 `_folderName`이 누락됨
3. 이후 프로세스에서 파일/폴더를 찾지 못해 대본 출력 실패

**해결책:**
v3.1.2의 저장 로직, `storyService.ts`, `formatScriptContent` 함수를 v3.2.0에 복원하면 문제가 해결됩니다.
