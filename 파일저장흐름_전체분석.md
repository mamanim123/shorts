## 🎯 현재 v3.2.0의 문제점

### 1. **대본 저장 불일치**

**서버 측:**
- ✅ `/api/generate/raw` 엔드포인트가 폴더를 자동 생성하고 파일을 저장함
- ✅ `_folderName`을 응답으로 전달함

**클라이언트 측:**
- ❌ `youtube-shorts-script-generator.tsx`에서 서버 저장을 추가로 호출하지 않음
- ❌ `processedScripts`를 바로 `setScripts()`로 설정

**결과:**
- 서버는 폴더를 생성하고 파일을 저장함 ✅
- 하지만 클라이언트가 추가 저장을 하지 않아 **데이터 불일치 발생 가능**
- `_folderName`이 프론트엔드 scripts 객체에 포함되지 않을 수 있음

### 2. **복잡한 저장 경로**

대본이 **2곳**에 저장됨:
1. `generated_scripts/[GEMINI]_날짜_제목.txt` (루트)
2. `generated_scripts/대본폴더/{제목}_{날짜}/[GEMINI]_날짜_제목.txt` (스토리 폴더)

이미지는 **1곳**에만 저장됨:
- `generated_scripts/images/{제목}_{날짜}/scene-01_...png`

### 3. **폴더명 매칭 불일치 가능성**

대본 폴더와 이미지 폴더의 `{제목}_{날짜}` 형식이 동일해야 하는데,  
다음 상황에서 불일치 발생 가능:
- 제목이 다르게 정규화될 경우
- 타임스탬프가 미묘하게 다를 경우

## 💡 해결 방안 제안

### 옵션 A: v3.1.2 방식으로 복원 (권장)

**v3.1.2의 작동 방식:**
1. 서버는 LLM 응답만 반환 (파일 저장 안 함)
2. 클라이언트에서 `saveStoryFile()` 호출하여 저장
3. 서버가 폴더 생성 + 파일 저장 + `folderName` 반환
4. 클라이언트가 반환된 `folderName`을 script 객체에 저장

**수정 파일:**
1. `services/storyService.ts` 복원
2. `youtube-shorts-script-generator.tsx`에 저장 로직 추가 (라인 1828 이전)

### 옵션 B: 현재 v3.2.0 구조 유지하되 개선

**수정 예시:**
```typescript
// youtube-shorts-script-generator.tsx 라인 1828
const scriptsWithFolder = processedScripts.map(script => ({
  ...script,
  _folderName: data._folderName  // ✅ 서버에서 받은 folderName 추가
}));
setScripts(scriptsWithFolder);
```

### 옵션 C: 저장 로직 완전 재설계

**새로운 구조:**
1. 대본 생성 시 서버가 UUID 기반 폴더 생성
2. 모든 관련 파일을 단일 스토리 폴더에 저장
3. `history.json`에 폴더 경로 참조만 저장
